# Frontend Design

Create distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Deliver real working code with exceptional attention to aesthetic details.

## Design Thinking

Before coding, assess:

- **Purpose** — What problem are we solving? Who is the audience?
- **Tone** — Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, Swiss precision, brutalist, tech-noir, warm analog, Japanese zen, Memphis, Bauhaus, neo-gothic, vaporwave, or something else true to the aesthetic direction.
- **Constraints** — Framework requirements, performance needs, accessibility standards.
- **Differentiation** — What's the memorable hook?

Bold maximalism and refined minimalism both work — the key is intentionality, not intensity.

## Frontend Aesthetics Guidelines

**Typography:** Choose fonts that are beautiful, unique, interesting. Avoid generic fonts like Arial and Inter. Pair a distinctive display font with a refined body font.

**Color & Theme:** Use CSS variables. Dominant colors with sharp accents outperform timid, evenly-distributed palettes.

**Motion:** CSS-only preferred for HTML; Motion library for React. Focus on high-impact moments — one well-orchestrated page load with staggered reveals. Use scroll-triggering and surprising hover states.

**Spatial Composition:** Unexpected layouts. Asymmetry. Overlap. Diagonal flow. Grid-breaking elements.

**Backgrounds & Visual Details:** Create atmosphere rather than defaulting to solid colors. Gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, grain overlays.

## Prohibited Patterns

- Overused fonts: Inter, Roboto, Arial
- Cliched color schemes (especially purple gradients on white backgrounds)
- Predictable centered layouts with hero-section-header-features-footer
- Cookie-cutter card grids with subtle shadows

## Implementation

Match complexity to vision. Maximalist designs need elaborate code with extensive animations. Minimalist designs need restraint and precision.

Vary output between light and dark themes, different font pairings, different aesthetics across tasks. No two designs should look the same.
