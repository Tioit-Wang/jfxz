# 金番写作 MVP 项目概述

**版本**：V1.3  
**文档类型**：项目概述

## 一、项目简介

**金番写作**是一款面向长篇内容创作场景的 AI 辅助写作工具。产品定位是“作者主导，AI 协助”：作者负责创作判断和内容取舍，AI 负责提供基于作品上下文的建议、分析和参考。

它的核心价值不在于“替作者写”，而在于帮助作者更稳定地推进写作、维护信息一致性，并降低与 AI 协作时反复补充背景信息的成本。

## 二、目标用户与核心痛点

**目标用户**：持续创作中长篇故事内容、需要长期维护作品信息、角色、设定和章节的作者。

**核心痛点**：

| 痛点                 | 表现                                                         |
| :------------------- | :----------------------------------------------------------- |
| **写作推进困难**     | 长时间停留在同一章节，难以决定下一段怎么写                   |
| **信息容易遗漏**     | 写到后期时，容易忘记角色、地点、规则和章节进展               |
| **风格不稳定**       | 不同时间写出的内容语气和质量不一致，后期修改成本高           |
| **通用 AI 上下文不足** | 每次和 AI 协作都要重新说明作品背景，影响写作节奏           |

## 三、产品核心设计理念

1. **辅助，而非替代**  
   AI 负责提供思路、方案和补充信息，但内容是否采用、如何修改，始终由作者决定。

2. **专注写作，减少干扰**  
   写作界面应尽量保持简单。帮助信息需要时出现，不需要时不打扰当前输入。

3. **基于作品上下文提供帮助**  
   AI 的价值来自对当前作品、角色、设定、章节和历史对话的理解，而不是泛泛回答。

4. **信息结构清晰，方便持续复用**  
   作品、角色、设定和章节分层明确，减少信息混杂和重复输入。

## 四、系统划分

当前系统分为两部分：

- **用户端**：面向作者使用，提供创作、信息管理、AI 对话、账户和套餐能力
- **管理端**：面向内部管理员使用，提供用户、模型、套餐、订单订阅、积分流水和会话的最小管理能力
  以及系统配置能力

## 五、用户端功能模块

- **作品**：管理作品基本信息和作品级总纲
- **角色**：单独管理作品中的人物信息
- **设定**：管理除角色外的地点、装备、属性、规则等设定
- **章节**：管理章节顺序、标题、正文和章节提要
- **编辑器**：提供纯文本写作区域和 AI 标记能力
- **智能对话面板**：基于当前作品上下文进行讨论、分析和建议生成
- **用户与认证**：管理注册、登录、账户和角色权限
- **套餐与计费**：管理 VIP 套餐、积分权益包、积分到账和积分消耗

## 六、管理端功能模块

- **用户管理**：查看用户、检索用户、禁用用户
- **模型管理**：维护可用 AI 模型、评分、上下文参数和计费倍率
- **套餐管理**：管理套餐和积分权益包商品
- **订单与订阅管理**：查看订单和订阅状态
- **积分流水**：查看积分发放、消耗、清零和调整记录
- **会话管理**：查看用户会话和排查问题
- **系统配置**：管理全局配置项，例如支付渠道配置

## 七、MVP 项目范围

**本期必做**：

- 用户端的作品、角色、设定、章节、编辑器、智能对话能力
- 用户注册、登录、基础账户能力
- 套餐与积分权益包的最小计费模型
- 管理端的用户管理、模型管理、套餐管理、订单订阅查看、积分流水查看、会话查看

**本期不做**：

- 独立故事线模块
- 多人协作与评论系统
- 对话导出、语音输入、图片上传等非核心能力
- 复杂运营后台和数据分析看板
