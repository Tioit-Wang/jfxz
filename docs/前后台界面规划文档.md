# 金番写作 MVP - 前后台界面规划文档

## 一、规划目的

本文档用于把现有需求文档转换成更容易落地的界面结构。

当前项目不适合把每个功能都拆成独立页面，尤其是用户端。用户端的核心场景是作者在同一本作品中持续写作、查看资料、调用 AI，因此更适合采用“大工作台 + 局部弹窗/覆盖层”的方式。

管理端则不同，管理端主要是列表查询、详情查看和状态管理，更适合采用“后台框架 + 左侧菜单 + 列表详情页”的方式。

## 二、整体规划原则

### 1. 用户端原则

用户端以创作体验为核心，尽量减少页面跳转。

- 作品内功能尽量集中在 `/books/:bookId` 一个大界面中。
- 章节、编辑器、AI 对话需要常驻展示，保证写作不中断。
- 作品信息、角色、设定、账户、套餐等低频功能使用大模态框承载。
- AI 建议不使用全屏弹窗，优先覆盖右侧对话区域，避免打断正文编辑。
- 路由只承担核心页面切换，不承担每个小功能的页面拆分。

### 2. 管理端原则

管理端以管理效率和信息检索为核心。

- 使用标准后台布局：顶部栏 + 左侧菜单 + 主内容区。
- 每个管理模块保留独立页面，方便管理员快速定位。
- MVP 阶段以查看、搜索、编辑基础状态为主，不做复杂运营看板。
- 详情信息可通过详情页、抽屉或大模态框展示。
- 管理端不追求沉浸式体验，优先清晰、稳定、可维护。

## 三、用户端界面规划

### 1. 用户端路由规划

| 路由 | 界面 | 说明 |
| :-- | :-- | :-- |
| `/` | 落地页 | 面向未登录用户展示产品介绍、核心卖点、使用入口和价格入口。 |
| `/books` | 作品列表页 | 展示当前用户创建的全部作品，支持新建、进入、删除作品；未登录时弹出登录注册模态框。 |
| `/books/:bookId` | 作品工作台页 | 用户端核心大界面，承载章节、编辑器、AI 对话和多个大模态框；未登录时弹出登录注册模态框。 |
| `/billing/pay/:orderId` | 支付页，可选 | 如果支付二维码不放在模态框中，可单独使用此页面。 |
| `/billing/success` | 支付成功页，可选 | 支付完成后的结果页，也可用模态框状态替代。 |

MVP 阶段建议优先实现 `/`、`/books`、`/books/:bookId` 三个核心路由。用户端不单独设置登录页和注册页，认证统一通过大模态框完成。支付相关页面可以根据实现复杂度决定是否独立拆出。

### 2. 用户端登录注册模态框

用户端不做独立登录页和注册页，登录与注册合并在同一个大模态框中。

触发场景：

- 用户在落地页点击“开始写作”“立即体验”等按钮。
- 未登录用户访问 `/books`。
- 未登录用户访问 `/books/:bookId`。
- 未登录用户触发需要账户身份的操作，例如创建作品、保存章节、发送 AI 对话、购买套餐。

交互规则：

- 模态框只保留一套邮箱登录入口。
- 用户提交邮箱后，如果账号已存在，则视为登录。
- 用户提交邮箱后，如果账号不存在，则自动创建账号并视为注册成功。
- 对用户来说不区分“登录”和“注册”两个动作，统一理解为“使用邮箱继续”。
- 登录注册成功后关闭模态框，并继续执行用户原本要做的操作。

这种方式可以降低新用户进入门槛，也避免在用户端维护两套认证页面。

### 3. `/books/:bookId` 工作台布局

`/books/:bookId` 是用户端最重要的界面，建议采用“上 + 下三栏”的布局。

```text
┌──────────────────────────────────────────────┐
│ 顶部栏：作品名 / 保存状态 / 积分 / 功能入口   │
├──────────────┬────────────────┬──────────────┤
│ 左侧章节列表 │ 中间正文编辑器 │ 右侧 AI 对话 │
│              │                │              │
└──────────────┴────────────────┴──────────────┘
```

### 4. 顶部栏

顶部栏负责展示当前作品上下文和低频功能入口。

建议包含：

- 当前作品名称。
- 返回作品列表。
- 当前章节保存状态，例如“保存中 / 已保存 / 保存失败”。
- 当前积分余额入口。
- 作品信息入口。
- 角色管理入口。
- 设定管理入口。
- 套餐与积分入口。
- 账户中心入口。

### 5. 左侧章节列表

左侧主要用于管理当前作品下的章节。

建议包含：

- 章节列表。
- 当前章节高亮。
- 章节顺序编号。
- 章节标题。
- 章节提要预览。
- 新建章节按钮。
- 删除章节入口。
- 调整章节顺序入口。

章节的正文编辑不在左侧完成，左侧只负责选择和管理章节结构。

### 6. 中间正文编辑器

中间区域是作者主要写作区域。

建议包含：

- 章节标题编辑。
- 纯文本正文编辑器。
- 章节提要编辑区，可放在正文上方、下方或折叠区域中。
- 本章字数。
- 今日字数。
- AI 分析按钮。
- AI 高亮段落。
- 高亮段落悬停提示气泡。

编辑器 MVP 不做富文本工具栏，不支持图片、表格和多章节同时编辑。

### 7. 右侧 AI 对话面板

右侧是固定智能对话区域，不建议独立成页面。

建议包含：

- 当前会话标题。
- 当前模型选择器。
- 新建会话按钮。
- 会话历史入口。
- 消息气泡流。
- 快捷指令按钮。
- 输入框。
- `@` 引用选择器。
- 引用块展示。
- 发送按钮。

AI 对话需要自动读取当前作品、当前章节，以及用户显式 `@` 的角色、设定、章节。

模型选择器只展示管理端启用模型，并展示模型名称、描述、逻辑评分、文笔评分、知识面评分、最大上下文和最大输出。用户端不展示 token 明细。

### 8. 用户端大模态框规划

作品内的低频管理功能建议通过大模态框完成。

| 模态框 | 入口 | 主要内容 |
| :-- | :-- | :-- |
| 作品信息模态框 | 顶部栏“作品信息” | 作品名称、短简介、梗概、题材标签、背景规则、重点要求、禁忌要求。 |
| 角色管理模态框 | 顶部栏“角色” | 角色列表、搜索、详情、新建、编辑、删除、复制。 |
| 设定管理模态框 | 顶部栏“设定” | 设定列表、类型筛选、搜索、详情、新建、编辑、删除、复制。 |
| 会话历史模态框 | 右侧对话面板 | 最近会话列表、新建会话、切换会话。 |
| 套餐与积分模态框 | 顶部栏“积分/套餐” | 当前套餐、VIP 每日积分、积分权益包余额、套餐购买、积分权益包购买、简化积分流水。 |
| 支付二维码模态框 | 套餐与积分模态框 | 展示支付宝二维码、支付状态、支付成功提示。 |
| 账户中心模态框 | 顶部栏头像/账户 | 邮箱、昵称、账户状态、当前套餐、退出登录。 |

### 9. 用户端覆盖层规划

AI 建议覆盖层建议覆盖在右侧对话面板的消息区上方，而不是覆盖整个页面。

触发方式：

- 用户点击编辑器中的 AI 高亮段落。

覆盖层内容：

- 原文引用。
- 问题描述。
- 一个或多个修改方案。
- “采纳替换”按钮。
- “发送至对话”按钮。
- 关闭按钮。

这样可以让用户一边保留正文上下文，一边处理 AI 建议。

## 四、管理端界面规划

### 1. 管理端整体布局

管理端建议使用标准后台布局。

```text
┌──────────────────────────────────────────────┐
│ 顶部栏：系统名 / 当前管理员 / 退出登录        │
├──────────────┬───────────────────────────────┤
│ 左侧菜单     │ 主内容区                       │
│              │ 列表 / 详情 / 表单 / 配置       │
└──────────────┴───────────────────────────────┘
```

### 2. 管理端路由规划

| 路由 | 界面 | 说明 |
| :-- | :-- | :-- |
| `/admin/login` | 管理员登录页 | 管理员登录入口，可与用户登录共用能力，但入口建议独立。 |
| `/admin` | 管理端首页 | MVP 可直接跳转到用户管理，不做数据看板。 |
| `/admin/users` | 用户管理页 | 用户列表、搜索、筛选、状态管理、用户详情。 |
| `/admin/models` | 模型管理页 | 模型列表、筛选、新建、编辑、启停、评分、上下文参数和计费倍率。 |
| `/admin/products` | 套餐与积分权益包管理页 | 套餐列表、积分权益包列表、筛选、新建、编辑、启停。 |
| `/admin/orders` | 订单管理页 | 订单列表、搜索、筛选、订单详情、支付记录入口。 |
| `/admin/subscriptions` | 订阅管理页 | 订阅列表、筛选、订阅详情、关联订单。 |
| `/admin/point-transactions` | 积分流水页 | 积分流水只读列表、筛选和详情。 |
| `/admin/sessions` | 会话管理页 | 会话列表、搜索、筛选、会话详情、历史记录只读查看。 |
| `/admin/configs` | 系统配置页 | 配置列表、分组筛选、配置编辑、密文字段掩码展示。 |

### 3. 用户管理页

主要能力：

- 用户列表。
- 按邮箱、昵称搜索。
- 按角色、账号状态、注册时间范围和最近登录时间范围筛选。
- 查看邮箱、昵称、角色、状态、注册时间、最近登录时间。
- 启用或禁用用户。
- 查看用户详情。
- 查看当前订阅状态和积分余额。

建议交互：

- 列表作为主界面。
- 用户详情使用右侧抽屉或详情模态框。
- 禁用用户需要二次确认。

### 4. 模型管理页

模型管理用于维护用户端智能对话和编辑器 AI 检查可用模型。

主要能力：

- 模型列表。
- 按状态、评分范围、上下文范围和最大输出范围筛选。
- 新增模型。
- 编辑模型。
- 启用或停用模型。
- 配置模型对外名称、平台调用 ID、描述、评分、上下文、最大输出和 temperature。
- 配置缓存命中输入倍率、缓存未命中输入倍率和输出倍率。
- 支持通过成本价和目标加价率快速生成倍率。

建议交互：

- 列表展示模型名称、平台调用 ID、状态、三项评分和计费倍率。
- 新建和编辑使用弹窗或抽屉。
- 停用模型需要二次确认。
- API key 不在页面展示或编辑。

### 5. 套餐与积分权益包管理页

该页面可以用一个页面内的两个 Tab 实现。

建议 Tab：

- 套餐管理。
- 积分权益包管理。

套餐管理包含：

- 套餐名称。
- 价格。
- VIP 每日积分。
- 附带积分权益包积分。
- 状态。
- 按状态、价格范围、每日 VIP 积分范围和套餐时长筛选。
- 新建套餐。
- 编辑套餐。
- 启用/停用。

积分权益包管理包含：

- 积分权益包名称。
- 价格。
- 积分数量。
- 状态。
- 按状态、价格范围和积分数量范围筛选。
- 新建积分权益包。
- 编辑积分权益包。
- 启用/停用。

### 6. 订单与订阅管理页

订单和订阅可以拆成两个页面，也可以在一个页面中用 Tab。考虑 MVP 清晰性，建议拆成两个页面。

订单管理页包含：

- 订单列表。
- 订单编号。
- 用户。
- 商品类型。
- 商品名称。
- 金额。
- 状态。
- 支付时间。
- 订单详情。
- 支付记录查看。
- 按用户、商品类型、订单状态、支付渠道、金额范围、下单时间范围和支付时间范围筛选。

订阅管理页包含：

- 订阅列表。
- 用户。
- 当前订阅套餐。
- 状态。
- 开始时间。
- 结束时间。
- 续费顺序。
- 订阅详情。
- 关联订单。
- 按用户、套餐、订阅状态、开始时间范围、结束时间范围和续费顺序筛选。

### 7. 积分流水页

积分流水页用于只读查看积分账务变动。

主要能力：

- 流水列表。
- 按用户、余额类型、变更类型、来源类型、模型、作品、积分变动范围和时间范围筛选。
- 查看积分变动、变动后余额、来源 ID。
- AI 消费流水展示模型快照、token usage、倍率快照和扣费积分。
- 用户端简化流水只展示时间、积分变动或消耗积分、关联作品；管理端完整流水展示排查字段。

MVP 不允许管理员手动新增、编辑或删除流水。

### 8. 会话管理页

会话管理用于排查问题，不用于运营编辑。

主要能力：

- 会话列表。
- 按用户、作品、会话标题搜索。
- 按用户、作品、会话来源和最近活跃时间范围筛选。
- 查看所属用户。
- 查看所属作品。
- 查看会话标题。
- 查看来源。
- 查看最近活跃时间。
- 查看会话详情。
- 只读查看历史消息。

MVP 不允许管理员手动编辑会话内容。

### 9. 系统配置页

系统配置主要用于管理支付等基础配置。

主要能力：

- 配置列表。
- 按分组查看。
- 按配置分组、值类型、是否必填和是否为空筛选。
- 展示配置键、类型、当前值、是否必填。
- 编辑配置值。
- secret 类型字段默认掩码展示。
- 支付宝当面付配置。
- 编辑器 AI 检查默认模型配置。

建议交互：

- 左侧为配置分组。
- 右侧为配置项列表。
- 编辑配置使用弹窗或抽屉。
- secret 字段需要明确“显示/隐藏”操作。
- 编辑器检查模型只能从启用模型中选择。

## 五、前后台差异总结

| 端 | 推荐形态 | 原因 |
| :-- | :-- | :-- |
| 用户端 | 大工作台 + 大模态框 + 右侧覆盖层 | 作者需要连续写作，频繁跳页会打断创作。 |
| 管理端 | 标准后台 + 左侧菜单 + 列表详情页 | 管理员主要做查询、查看、配置和状态管理。 |

## 八、当前推荐方案

用户端采用：

```text
/books/:bookId = 一个完整创作工作台
顶部栏 + 左侧章节 + 中间编辑器 + 右侧 AI 对话 + 大模态框
```

管理端采用：

```text
/admin = 一个标准后台系统
顶部栏 + 左侧菜单 + 主内容列表页 + 详情抽屉/弹窗
```

这种规划能兼顾 MVP 开发效率和后续扩展性。用户端保证创作体验，管理端保证信息管理效率。
