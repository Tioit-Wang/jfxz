# 文档三：妙蛙写作 MVP - 作品需求文档 V1.0

**模块定位**：作品是系统中的顶层对象，用于承载一本书的基础信息和总纲信息。  
**核心原则**：作品信息一次定义，后续由角色、设定、章节和对话能力持续复用。

## 一、作品基础能力

| 编号   | 功能         | 描述                                                     | 交互说明                                                                 |
| :----- | :----------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **W1** | 作品列表     | 展示当前用户创建的全部作品。                             | 每项至少显示作品名称、短简介、最近更新时间。                             |
| **W2** | 新建作品     | 用户可以创建新作品。                                     | 创建时填写基础字段，保存后进入该作品工作台。                             |
| **W3** | 编辑作品     | 用户可以修改作品的基础信息和总纲信息。                   | 编辑后保存，作品详情和相关上下文同步更新。                               |
| **W4** | 删除作品     | 用户可以删除不再需要的作品。                             | 删除前需要二次确认，删除后其下角色、设定、章节数据一并删除。             |
| **W5** | 选择当前作品 | 进入某个作品后，后续角色、设定、章节、对话都绑定该作品。 | 页面顶部需持续显示当前作品名称，避免跨作品误操作。                       |

## 二、作品字段定义

作品本身直接包含“基础信息 + 总纲信息”，不再单独拆出独立的大纲模块。

| 字段名             | 必填 | 描述                                                                 |
| :----------------- | :--- | :------------------------------------------------------------------- |
| **作品名称**       | 是   | 作品的主标题。                                                       |
| **短简介**         | 是   | 用于快速介绍这本书的简短说明，长度短于梗概。                         |
| **梗概**           | 是   | 用于完整说明这本书到底在讲什么，是作品级的核心总纲。                 |
| **题材标签**       | 是   | 用于标识作品类型和读者预期，如都市、穿越、年代、悬疑等。             |
| **背景与世界规则** | 是   | 用于描述故事发生环境、时代背景、世界设定和需要遵守的规则。           |
| **重点要求**       | 否   | 用于说明这本书需要重点强调、重点书写的方向。                         |
| **禁忌要求**       | 否   | 用于说明这本书不能出现或应尽量避免的内容。                           |

## 三、作品页面展示

| 编号   | 功能           | 描述                                                         | 交互说明                                                                 |
| :----- | :------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **W6** | 作品详情展示   | 在作品详情页中展示全部作品字段。                             | 短简介适合概览展示；梗概和背景与世界规则支持大段文本查看。               |
| **W7** | 标签展示       | 题材标签以标签形式展示。                                     | 支持多标签展示，便于快速识别作品定位。                                   |
| **W8** | 作品信息复用   | 当前作品的名称、梗概、背景与世界规则可被其他模块读取。       | 编辑器分析、对话上下文注入和角色/设定创建时都可引用当前作品信息。       |

## 四、与 AI 的联动能力

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **W9** | 对话上下文注入   | 对话面板在当前作品下默认读取作品名称、梗概、题材标签和背景信息。     | 用户无需每次重复说明作品是什么类型、发生在什么背景下。                   |
| **W10** | 作品信息辅助生成 | AI 可辅助生成短简介、梗概草稿或背景描述草稿。                        | 生成结果由用户确认后再写回作品字段。                                     |
| **W11** | 作品约束生效     | 重点要求和禁忌要求会作为 AI 对话和分析的重要约束。                   | 目标是减少偏题、跑题和与作品定位不符的建议。                             |

## 五、MVP 边界（作品部分）

- 不支持分卷结构。
- 不支持同一作品下维护多个版本的总纲。
- 不支持故事线模块。
- 不支持作品间复制角色、设定或章节。

## 六、验收标准（作品部分）

1. 用户可创建、编辑、删除作品。
2. 作品表单可保存作品名称、短简介、梗概、题材标签、背景与世界规则、重点要求、禁忌要求。
3. 作品详情页可完整展示上述字段。
4. 当前作品切换后，角色、设定、章节和对话上下文同步切换。
5. 对话能力可以读取作品梗概、背景与世界规则、重点要求和禁忌要求。
