# 文档八：金番写作 MVP - 套餐与计费需求文档 V1.0

**模块定位**：定义用户端的套餐、加油包、积分到账和积分扣减规则。  
**核心原则**：对用户按“积分”计费，对内按模型 token 成本核算；套餐结构清晰，后端实现尽量简化。

## 一、商品模型

当前系统有两类可售商品：

1. **套餐**
   套餐 = 月度积分 + 附带加油包积分

2. **加油包**
   加油包可单独售卖，购买后直接增加加油包积分

## 二、套餐规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P1** | 套餐展示         | 用户可查看当前可购买套餐。                                           | 每个套餐至少展示价格、月度积分、附带加油包积分。                         |
| **P2** | 套餐购买         | 用户购买套餐后，立即获得对应套餐权益。                               | 权益由两部分构成：月度积分、附带加油包积分。                             |
| **P3** | 套餐续费         | 套餐按月订阅。                                                       | 每个账期开始时重新发放该套餐的月度积分和附带加油包积分。                 |
| **P4** | 月度积分规则     | 月度积分按账期发放，不按天拆分发放。                                 | 当前账期结束后未使用的月度积分失效，不结转。                             |
| **P5** | 附带加油包规则   | 套餐附带加油包积分在套餐购买或续费成功后立即到账。                   | MVP 阶段建议与当前订阅周期同步到期，以简化实现。                         |

## 三、加油包规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P6** | 加油包展示       | 用户可查看当前可单独购买的加油包。                                   | 每个加油包至少展示价格、积分数量、有效期。                               |
| **P7** | 加油包购买       | 用户购买加油包后，积分立即到账。                                     | 记入加油包积分余额。                                                     |
| **P8** | 加油包有效期     | 单独购买的加油包支持独立有效期。                                     | 有效期建议由商品配置，例如 30 / 90 / 365 天。                            |

## 四、积分账户与扣减规则

MVP 阶段用户侧只区分两类积分余额：

- **月度积分**
- **加油包积分**

其中：

- 套餐中的附带加油包积分，记入“加油包积分”
- 单独购买的加油包积分，也记入“加油包积分”

| 编号    | 功能             | 描述                                                         | 交互说明                                                                 |
| :------ | :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P9**  | 积分余额展示     | 用户可查看当前总积分、月度积分和加油包积分。                 | 可在个人中心或套餐页展示。                                               |
| **P10** | 扣减顺序         | AI 能力消耗积分时，优先扣减月度积分，再扣减加油包积分。      | 这样可以优先消耗更容易到期的积分。                                       |
| **P11** | 消耗场景         | 对话、编辑器分析等 AI 能力都会扣减积分。                     | 用户侧不直接看到 token，只看到积分消耗。                                 |

积分余额和积分流水均保留 2 位小数。用户端仅展示余额，不展示 token 明细和积分流水明细。

## 五、Token 计费与扣费计划

MVP 阶段采用以下原则：

- 用户端不直接按 token 计费
- 用户端统一按积分消费
- 不做“每天发多少积分，今日不用就过期”的规则
- 套餐按月发放月度积分，并附带一次性到账的加油包积分

### 5.1 基准计费规则

- 模型供应商地址、请求协议、API key 管理和调用 ID 规则统一见 `../管理端需求文档/模型管理需求文档.md`。
- 基准价：`10 元 / 100万 token`。
- 积分面值：`1 积分 = 0.1 元`。
- 基准积分单价：`100 积分 / 100万 token`。
- 扣费积分保留 2 位小数。
- 模型倍率保留 2 位小数并向上取整。

### 5.2 扣费公式

```text
扣费积分 =
  prompt_cache_hit_tokens / 1000000 * 100 * cache_hit_input_multiplier
+ prompt_cache_miss_tokens / 1000000 * 100 * cache_miss_input_multiplier
+ completion_tokens / 1000000 * 100 * output_multiplier
```

说明：

- `prompt_cache_hit_tokens` 表示缓存命中输入 token。
- `prompt_cache_miss_tokens` 表示缓存未命中输入 token。
- `completion_tokens` 表示输出 token。
- `reasoning_tokens` 已包含在 `completion_tokens` 中，不重复计费。
- 若供应商未返回缓存拆分字段，则输入 token 默认按缓存未命中计费。

### 5.3 扣费时机与失败规则

- AI 调用完成并取得有效结果后，按真实 usage 实扣积分。
- 调用前只要求用户总积分余额大于 0。
- 模型未配置、模型停用、请求失败、解析失败或无有效结果时，不扣积分。
- AI 扣费优先扣月度积分，再扣加油包积分。
- 一次 AI 扣费可拆成多条积分流水：先扣月度积分，不足部分再扣加油包积分。
- 若剩余扣费超过当前加油包积分，加油包积分可在本次消费流水中临时扣为负数，随后系统追加一条“系统赠送”流水，将加油包积分和总余额补平到 0。
- 系统赠送只用于补平单次 AI 调用超额扣费，不作为套餐权益、营销赠送或用户可主动领取的积分。

### 5.4 模型默认计费倍率

| 模型 | 缓存命中输入倍率 | 缓存未命中输入倍率 | 输出倍率 | 说明 |
| :--- | :--------------- | :----------------- | :------- | :--- |
| `DeepSeek-v4-flash` | `0.11` | `0.11` | `0.22` | 输入成本 `1 元 / 100万 token`、输出成本 `2 元 / 100万 token`，按 10% 毛利生成。 |
| `DeepSeek-v4-pro` | `0.02` | `1.32` | `2.64` | 缓存命中输入成本 `0.10 元 / 100万 token`，未命中输入成本 `12 元 / 100万 token`，输出成本 `24 元 / 100万 token`，按 10% 毛利生成。 |

## 六、支付方式

MVP 阶段当前只接入一个支付渠道：

- **支付宝当面付（扫码支付）**

支付流程采用：

1. 用户创建套餐或加油包订单
2. 服务端调用支付宝当面付预下单接口生成二维码
3. 前端展示二维码
4. 用户扫码支付
5. 支付成功后发放对应积分或订阅权益

详细支付方案见 `../支付接入需求文档.md`。

## 七、MVP 边界（套餐与计费）

- 不支持优惠券、折扣码。
- 不支持发票和复杂财务对账。
- 不支持积分转赠。
- 不支持手动拆分账期或补发每日积分。
- 不支持多支付渠道。
- 不支持自动退款。
- 用户端不展示积分流水明细。

## 八、验收标准（套餐与计费）

1. 用户可查看套餐和加油包商品。
2. 用户购买套餐后，能同时获得月度积分和附带加油包积分。
3. 用户可单独购买加油包。
4. 用户可查看月度积分和加油包积分余额。
5. AI 消耗积分时，先扣月度积分，再扣加油包积分。
6. 用户可通过支付宝扫码完成套餐或加油包支付。
7. AI 调用成功后，可按 OpenAI-compatible usage 计算并扣减 2 位小数积分。
8. AI 调用失败、解析失败、模型未配置、模型停用或无有效结果时不扣积分。
9. AI 扣费后余额低于 0 时，系统通过“系统赠送”流水补平到 0。
