# 文档八：金番写作 MVP - 套餐与计费需求文档 V2.0

**模块定位**：定义用户端的套餐、字数包、积分到账和积分扣减规则。
**核心原则**：对用户按"积分"计费，对内按模型 token 成本核算；套餐结构清晰，后端实现尽量简化。

## 一、商品模型

当前系统有两类可售商品：

1. **套餐（VIP）**
   套餐 = VIP福利计划 + 附带字数包

   - **VIP福利计划**：每天发放 N 个积分，当日未使用部分在次日凌辰5:00过期。在套餐时间内每天都发放。套餐固定 31 天，共发放 31 次（含购买当天）。不允许叠加购买，必须到期后才能再买。
   - **附带字数包**：购买套餐时一次性发放固定积分，有效期 1 年（从购买日起算），与 VIP 福利计划完全独立。

2. **字数包（独立商品）**
   可单独购买，购买后积分立即到账。支持多次购买，积分累加但**有效期各自独立**计算。

**关键规则：**
- 用户已购买的套餐权益在购买时锁定，不受管理员后续修改套餐配置的影响。
- 字数包与 VIP 套餐完全独立——VIP 过期后字数包积分仍可继续使用直到自己的有效期到期。
- 无论何时购买套餐，都发放全额当日积分（次日凌晨5:00过期）。

## 二、套餐规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P1** | 套餐展示         | 用户可查看当前可购买的所有套餐。                                     | 每个套餐至少展示：名称、价格、每日积分数、附带字数包积分数、套餐时长（31天）。 |
| **P2** | 套餐购买         | 用户购买套餐后，立即获得当日VIP积分 + 附带字数包积分。               | 购买时锁定当前套餐配置，后续管理员修改不影响已购买权益。已有未到期套餐时不允许再次购买。 |
| **P3** | VIP每日积分发放  | 每天凌晨5:00自动发放当日积分，同时过期前一日未使用的积分。           | 套餐31天内共发放31次，最后一次发放的积分在第32天凌晨5:00过期。          |
| **P4** | 附带字数包规则   | 套餐附带字数包积分在购买成功后立即到账，有效期1年。                  | 与VIP福利计划完全独立，VIP过期不影响字数包积分的使用。                   |

## 三、字数包规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P5** | 字数包展示       | 用户可查看当前可单独购买的字数包。                                   | 每个字数包至少展示：名称、价格、积分数量、有效期。                       |
| **P6** | 字数包购买       | 用户购买字数包后，积分立即到账。                                     | 无VIP套餐也可购买。支持多次购买，各批次有效期独立。                      |
| **P7** | 字数包有效期     | 每次购买的字数包独立计算有效期。                                     | 有效期从购买日起算，天数由商品配置决定（如 365 天）。                    |
| **P8** | 字数包展示层级   | 常规界面只展示字数包总余额。                                         | 详情页展示各批次列表（积分数量、到期日）。                               |

## 四、积分账户与扣减规则

用户侧区分两类积分余额：

- **VIP每日积分**：当天发放，次日凌辰5:00过期未用部分。
- **字数包积分**：多条购买记录，各批次独立有效期。

| 编号    | 功能             | 描述                                                         | 交互说明                                                                 |
| :------ | :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P9**  | 积分余额展示     | 用户可查看：VIP每日剩余积分、字数包总积分、总积分。           | 可在个人中心或套餐页展示。                                               |
| **P10** | 扣减顺序         | AI 能力消耗积分时，优先扣减VIP每日积分，再扣减字数包积分。    | 字数包积分按到期日从早到晚依次扣减（FIFO），优先消耗最早到期的积分。     |
| **P11** | 消耗场景         | 对话、编辑器分析等 AI 能力都会扣减积分。                     | 用户侧不直接看到 token，只看到积分消耗。                                 |
| **P12** | 当前套餐状态     | 用户可查看当前VIP套餐的生效时间、到期时间、今日剩余积分。     | 无套餐时展示"未购买"状态。                                              |

积分余额和积分流水均保留 2 位小数。用户端仅展示余额，不展示 token 明细和积分流水明细。

**扣减示例：**

假设用户VIP每日额度100积分，拥有两条字数包记录（A: 500积分，3月1日到期；B: 300积分，6月1日到期）。用户本次AI调用需扣200积分：

1. 先扣VIP每日积分100（剩余0）
2. 再扣字数包A的100积分（A剩余400）
3. 最终：VIP当日额度=0，字数包A=400，字数包B=300

## 五、Token 计费与扣费计划

MVP 阶段采用以下原则：

- 用户端不直接按 token 计费
- 用户端统一按积分消费
- 模型供应商地址、请求协议、API key 管理和调用 ID 规则统一见 `../管理端需求文档/模型管理需求文档.md`。

### 5.1 基准计费规则

- 基准价：`10 元 / 100万 token`。
- 积分面值：`1 积分 = 0.1 元`。
- 基准积分单价：`100 积分 / 100万 token`。
- 扣费积分保留 2 位小数。
- 模型倍率保留 2 位小数并向上取整。

### 5.2 扣费公式

```text
扣费积分 =
  prompt_cache_hit_tokens / 1000000 * 100 * cache_hit_input_multiplier
+ prompt_cache_miss_tokens / 1000000 * 100 * cache_miss_input_multiplier
+ completion_tokens / 1000000 * 100 * output_multiplier
```

说明：

- `prompt_cache_hit_tokens` 表示缓存命中输入 token。
- `prompt_cache_miss_tokens` 表示缓存未命中输入 token。
- `completion_tokens` 表示输出 token。
- `reasoning_tokens` 已包含在 `completion_tokens` 中，不重复计费。
- 若供应商未返回缓存拆分字段，则输入 token 默认按缓存未命中计费。

### 5.3 扣费时机与失败规则

- AI 调用完成并取得有效结果后，按真实 usage 实扣积分。
- 调用前只要求用户总积分余额大于 0。
- 模型未配置、模型停用、请求失败、解析失败或无有效结果时，不扣积分。
- AI 扣费优先扣VIP每日积分，再扣字数包积分（FIFO）。
- 一次 AI 扣费可拆成多条积分流水：先扣VIP每日积分，不足部分再扣字数包积分。
- 若剩余扣费超过当前字数包积分，字数包积分可在本次消费流水中临时扣为负数，随后系统追加一条"系统赠送"流水，将字数包积分和总余额补平到 0。
- 系统赠送只用于补平单次 AI 调用超额扣费，不作为套餐权益、营销赠送或用户可主动领取的积分。

### 5.4 模型默认计费倍率

| 模型 | 缓存命中输入倍率 | 缓存未命中输入倍率 | 输出倍率 | 说明 |
| :--- | :--------------- | :----------------- | :------- | :--- |
| `DeepSeek-v4-flash` | `0.11` | `0.11` | `0.22` | 输入成本 `1 元 / 100万 token`、输出成本 `2 元 / 100万 token`，按 10% 毛利生成。 |
| `DeepSeek-v4-pro` | `0.02` | `1.32` | `2.64` | 缓存命中输入成本 `0.10 元 / 100万 token`，未命中输入成本 `12 元 / 100万 token`，输出成本 `24 元 / 100万 token`，按 10% 毛利生成。 |

## 六、支付方式

MVP 阶段当前只接入一个支付渠道：

- **支付宝当面付（扫码支付）**

支付流程采用：

1. 用户创建套餐或字数包订单
2. 服务端调用支付宝当面付预下单接口生成二维码
3. 前端展示二维码
4. 用户扫码支付
5. 支付成功后发放对应积分或订阅权益

详细支付方案见 `../支付接入需求文档.md`。

## 七、MVP 边界（套餐与计费）

- 不支持优惠券、折扣码。
- 不支持限时活动、分渠道定价、复杂促销组合。
- 不支持发票和复杂财务对账。
- 不支持积分转赠。
- 不支持自动退款。
- 不支持多支付渠道。
- 不支持套餐升级/降级（到期后重新购买即可）。
- 用户端不展示积分流水明细。

## 八、验收标准（套餐与计费）

1. 用户可查看套餐和字数包商品。
2. 用户购买套餐后，能同时获得当日VIP积分和附带字数包积分。
3. 用户可单独购买字数包，积分立即到账。
4. 用户可查看VIP每日积分和字数包积分余额。
5. 字数包详情页展示各批次列表（积分数量、到期日）。
6. AI 消耗积分时，先扣VIP每日积分，再扣字数包积分（FIFO）。
7. 用户可通过支付宝扫码完成套餐或字数包支付。
8. 已购买的套餐权益不受管理员后续修改的影响。
9. VIP套餐过期后，字数包积分仍可继续使用。
10. AI 调用成功后，可按 OpenAI-compatible usage 计算并扣减 2 位小数积分。
11. AI 调用失败、解析失败、模型未配置、模型停用或无有效结果时不扣积分。
12. AI 扣费后余额低于 0 时，系统通过"系统赠送"流水补平到 0。
