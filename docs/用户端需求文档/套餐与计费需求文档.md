# 文档八：金番写作 MVP - 套餐与计费需求文档 V3.0

**模块定位**：定义用户端的 VIP 套餐、积分权益包、积分到账、积分扣减和用户流水规则。
**核心原则**：对用户统一按积分消费，对内按模型 token 成本核算；VIP 每日积分用于订阅权益，积分权益包用于长期余额补充。

## 一、商品模型

当前系统有两类可售商品：

1. **套餐（VIP）**
   套餐 = VIP每日积分计划 + 附带积分权益包。

   - **VIP每日积分计划**：套餐固定 31 天。套餐生效期内每天 5:00 发放 N 个 VIP 每日积分，并清零前一日未使用的 VIP 每日积分。
   - **附带积分权益包**：购买套餐时一次性发放固定积分，永久有效，直接叠加到积分权益包余额。

2. **积分权益包（独立商品）**
   可单独购买，购买后积分立即到账，永久有效。支持多次购买，积分直接叠加到积分权益包余额。

**关键规则：**

- 用户已购买的套餐和积分权益包权益在购买时锁定，不受管理员后续修改商品配置影响。
- VIP 每日积分与积分权益包余额完全独立。
- VIP 每日积分每天清零，积分权益包余额永久有效。
- AI 消耗积分时，优先扣 VIP 每日积分，再扣积分权益包余额。

## 二、套餐规则

| 编号 | 功能 | 描述 | 交互说明 |
| :--- | :--- | :--- | :--- |
| **P1** | 套餐展示 | 用户可查看当前可购买的所有套餐。 | 每个套餐至少展示：名称、价格、每日积分数、附带积分权益包积分数、套餐时长 31 天。 |
| **P2** | 首次购买 | 用户购买套餐后，立即获得当日 VIP 每日积分，并获得附带积分权益包积分。 | 购买时锁定当前套餐配置。 |
| **P3** | VIP 每日积分发放 | 每天 5:00 自动发放当日 VIP 每日积分，同时清零前一日未使用的 VIP 每日积分。 | 套餐 31 天内共发放 31 次，最后一次发放的积分在第 32 天 5:00 清零。 |
| **P4** | 附带积分权益包 | 套餐附带积分权益包在支付成功后立即到账，永久有效。 | VIP 到期不影响积分权益包余额。 |
| **P5** | 套餐续费 | MVP 支持在 VIP 未到期时继续购买套餐。 | 新购买套餐从当前 VIP 到期时间后顺延生效；不支持立即切换权益。 |
| **P6** | 升级降级限制 | MVP 不支持套餐升级、降级或按差价切换。 | 用户只能购买新套餐形成顺延订阅。 |

续费说明：

- 若用户没有有效 VIP，支付成功后套餐立即生效。
- 若用户已有未到期 VIP，支付成功后创建一段新的订阅周期，`start_at` 等于当前有效 VIP 的 `end_at`，并从该时点开始计算新的 31 次发放周期。
- VIP 订阅的 `end_at` 表示该周期第 31 次发放积分在第 32 天 5:00 清零的时点。
- 续费套餐的每日积分数以购买时锁定的商品快照为准。
- 续费套餐附带的积分权益包仍在支付成功后立即发放，不等待新订阅周期开始。

## 三、积分权益包规则

| 编号 | 功能 | 描述 | 交互说明 |
| :--- | :--- | :--- | :--- |
| **P7** | 积分权益包展示 | 用户可查看当前可单独购买的积分权益包。 | 每个积分权益包至少展示：名称、价格、积分数量。 |
| **P8** | 积分权益包购买 | 用户购买积分权益包后，积分立即到账。 | 无 VIP 套餐也可购买。支持多次购买，余额直接叠加。 |
| **P9** | 永久有效 | 积分权益包不设置有效期。 | 不需要批次到期、FIFO 扣减或过期任务。 |

## 四、积分账户与扣减规则

用户侧区分两类积分余额：

- **VIP每日积分**：当天发放，次日 5:00 清零未使用部分。
- **积分权益包余额**：购买或套餐附带获得，永久有效，直接叠加。

| 编号 | 功能 | 描述 | 交互说明 |
| :--- | :--- | :--- | :--- |
| **P10** | 积分余额展示 | 用户可查看：VIP 每日剩余积分、积分权益包余额、总积分。 | 可在个人中心、套餐页或顶部栏入口展示。 |
| **P11** | 扣减顺序 | AI 能力消耗积分时，优先扣 VIP 每日积分，再扣积分权益包余额。 | 积分权益包不区分批次。 |
| **P12** | 消耗场景 | 对话、编辑器分析等 AI 能力都会扣减积分。 | 用户侧不直接看到 token。 |
| **P13** | 当前套餐状态 | 用户可查看当前 VIP 套餐的生效时间、到期时间、今日剩余积分和后续续费周期。 | 无套餐时展示“未购买”状态。 |

积分余额和积分流水均保留 2 位小数。

**扣减示例：**

假设用户 VIP 每日积分余额为 100，积分权益包余额为 500。本次 AI 调用真实扣费 200 积分：

1. 先扣 VIP 每日积分 100，VIP 每日积分余额变为 0。
2. 再扣积分权益包余额 100，积分权益包余额变为 400。
3. 总积分余额变为 400。

## 五、Token 计费与扣费计划

MVP 阶段采用以下原则：

- 用户端不直接按 token 计费。
- 用户端统一按积分消费。
- 模型供应商地址、请求协议、API key 管理和调用 ID 规则统一见 `../管理端需求文档/模型管理需求文档.md`。

### 5.1 基准计费规则

- 基准价：`10 元 / 100万 token`。
- 积分面值：`1 积分 = 0.1 元`。
- 基准积分单价：`100 积分 / 100万 token`。
- 扣费积分保留 2 位小数。
- 模型倍率保留 2 位小数并向上取整。

### 5.2 扣费公式

```text
扣费积分 =
  prompt_cache_hit_tokens / 1000000 * 100 * cache_hit_input_multiplier
+ prompt_cache_miss_tokens / 1000000 * 100 * cache_miss_input_multiplier
+ completion_tokens / 1000000 * 100 * output_multiplier
```

说明：

- `prompt_cache_hit_tokens` 表示缓存命中输入 token。
- `prompt_cache_miss_tokens` 表示缓存未命中输入 token。
- `completion_tokens` 表示输出 token。
- `reasoning_tokens` 已包含在 `completion_tokens` 中，不重复计费。
- 若供应商未返回缓存拆分字段，则输入 token 默认按缓存未命中计费。

### 5.3 调用前余额预检

AI 调用前不再只判断用户总积分是否大于 0。服务端必须按以下方式预估本次最高积分消耗：

```text
预估最高积分 =
  预估输入 token / 1000000 * 100 * 对应输入倍率
+ 当前模型最大输出 token / 1000000 * 100 * 输出倍率
```

规则：

- 预估输入 token 由服务端根据本次用户输入、引用上下文、作品信息、最近历史和系统提示词估算。
- 若无法拆分缓存命中与未命中输入 token，预检时全部按缓存未命中输入倍率估算。
- 当前模型最大输出 token 使用管理端模型配置中的 `max_output_tokens`。
- 若用户总积分余额小于预估最高积分，服务端拒绝本次调用，并提示用户购买套餐或积分权益包。
- 预检只用于控制成本风险；调用成功后仍按供应商返回的真实 usage 实扣。

### 5.4 扣费时机与失败规则

- AI 调用完成并取得有效结果后，按真实 usage 实扣积分。
- 模型未配置、模型停用、余额预检未通过、请求失败、解析失败或无有效结果时，不扣积分。
- AI 扣费优先扣 VIP 每日积分，再扣积分权益包余额。
- 一次 AI 扣费可拆成多条积分流水：先扣 VIP 每日积分，不足部分再扣积分权益包余额。
- 正常扣费流程不允许扣出负数。

### 5.5 模型默认计费倍率

| 模型 | 缓存命中输入倍率 | 缓存未命中输入倍率 | 输出倍率 | 说明 |
| :--- | :--------------- | :----------------- | :------- | :--- |
| `DeepSeek-v4-flash` | `0.11` | `0.11` | `0.22` | 输入成本 `1 元 / 100万 token`、输出成本 `2 元 / 100万 token`，按 10% 加价率生成。 |
| `DeepSeek-v4-pro` | `0.02` | `1.32` | `2.64` | 缓存命中输入成本 `0.10 元 / 100万 token`，未命中输入成本 `12 元 / 100万 token`，输出成本 `24 元 / 100万 token`，按 10% 加价率生成。 |

## 六、用户端积分流水

用户端提供简化积分流水，只用于让用户理解自己的积分变化。

| 编号 | 功能 | 描述 | 交互说明 |
| :--- | :--- | :--- | :--- |
| **P14** | 流水列表 | 用户可查看自己的积分流水。 | 只展示时间、积分变动或消耗积分、关联作品。 |
| **P15** | 关联作品 | AI 对话和编辑器分析产生的消费流水需展示关联作品。 | 非作品来源的发放或调整流水可展示为空或“系统”。 |
| **P16** | 明细边界 | 用户端不展示 token、模型倍率、模型调用 ID、成本、后台来源 ID。 | 完整排查信息仅管理端可见。 |

## 七、支付方式

MVP 阶段当前只接入一个支付渠道：

- **支付宝当面付（扫码支付）**

支付流程采用：

1. 用户创建套餐或积分权益包订单。
2. 服务端调用支付宝当面付预下单接口生成二维码。
3. 前端展示二维码。
4. 用户扫码支付。
5. 支付成功后发放对应积分或订阅权益。

详细支付方案见 `../支付接入需求文档.md`。

## 八、MVP 边界（套餐与计费）

- 不支持优惠券、折扣码。
- 不支持限时活动、分渠道定价、复杂促销组合。
- 不支持发票和复杂财务对账。
- 不支持积分转赠。
- 不支持自动退款。
- 不支持多支付渠道。
- 不支持套餐升级、降级或按差价切换。
- 不支持积分权益包过期、批次列表或 FIFO 扣减。
- 用户端不展示 token 明细、模型倍率和后台排查字段。

## 九、验收标准（套餐与计费）

1. 用户可查看套餐和积分权益包商品。
2. 用户购买套餐后，能获得当日 VIP 每日积分和附带积分权益包积分。
3. 用户可在 VIP 未到期时续费，新订阅周期按当前到期时间顺延。
4. 用户可单独购买积分权益包，积分立即到账且永久有效。
5. 用户可查看 VIP 每日积分、积分权益包余额和总积分。
6. AI 调用前服务端可按模型最大输出和本次上下文估算预检余额。
7. 余额预检不通过时，AI 调用被拒绝且不扣积分。
8. AI 消耗积分时，先扣 VIP 每日积分，再扣积分权益包余额。
9. 用户可通过支付宝扫码完成套餐或积分权益包支付。
10. 已购买的套餐和积分权益包权益不受管理员后续修改影响。
11. VIP 套餐到期后，积分权益包余额仍可继续使用。
12. AI 调用成功后，可按 OpenAI-compatible usage 计算并扣减 2 位小数积分。
13. AI 调用失败、解析失败、模型未配置、模型停用、余额预检未通过或无有效结果时不扣积分。
14. 用户可查看简化积分流水，字段只包含时间、积分变动或消耗积分、关联作品。
