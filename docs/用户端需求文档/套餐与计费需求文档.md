# 文档八：金番写作 MVP - 套餐与计费需求文档 V1.0

**模块定位**：定义用户端的套餐、加油包、积分到账和积分扣减规则。  
**核心原则**：对用户按“积分”计费，对内按模型 token 成本核算；套餐结构清晰，后端实现尽量简化。

## 一、商品模型

当前系统有两类可售商品：

1. **套餐**
   套餐 = 月度积分 + 附带加油包积分

2. **加油包**
   加油包可单独售卖，购买后直接增加加油包积分

## 二、套餐规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P1** | 套餐展示         | 用户可查看当前可购买套餐。                                           | 每个套餐至少展示价格、月度积分、附带加油包积分。                         |
| **P2** | 套餐购买         | 用户购买套餐后，立即获得对应套餐权益。                               | 权益由两部分构成：月度积分、附带加油包积分。                             |
| **P3** | 套餐续费         | 套餐按月订阅。                                                       | 每个账期开始时重新发放该套餐的月度积分和附带加油包积分。                 |
| **P4** | 月度积分规则     | 月度积分按账期发放，不按天拆分发放。                                 | 当前账期结束后未使用的月度积分失效，不结转。                             |
| **P5** | 附带加油包规则   | 套餐附带加油包积分在套餐购买或续费成功后立即到账。                   | MVP 阶段建议与当前订阅周期同步到期，以简化实现。                         |

## 三、加油包规则

| 编号   | 功能             | 描述                                                                 | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P6** | 加油包展示       | 用户可查看当前可单独购买的加油包。                                   | 每个加油包至少展示价格、积分数量、有效期。                               |
| **P7** | 加油包购买       | 用户购买加油包后，积分立即到账。                                     | 记入加油包积分余额。                                                     |
| **P8** | 加油包有效期     | 单独购买的加油包支持独立有效期。                                     | 有效期建议由商品配置，例如 30 / 90 / 365 天。                            |

## 四、积分账户与扣减规则

MVP 阶段用户侧只区分两类积分余额：

- **月度积分**
- **加油包积分**

其中：

- 套餐中的附带加油包积分，记入“加油包积分”
- 单独购买的加油包积分，也记入“加油包积分”

| 编号    | 功能             | 描述                                                         | 交互说明                                                                 |
| :------ | :--------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **P9**  | 积分余额展示     | 用户可查看当前总积分、月度积分和加油包积分。                 | 可在个人中心或套餐页展示。                                               |
| **P10** | 扣减顺序         | AI 能力消耗积分时，优先扣减月度积分，再扣减加油包积分。      | 这样可以优先消耗更容易到期的积分。                                       |
| **P11** | 消耗场景         | 对话、编辑器分析等 AI 能力都会扣减积分。                     | 用户侧不直接看到 token，只看到积分消耗。                                 |

## 五、MVP 计费策略

MVP 阶段采用以下原则：

- 用户端不直接按 token 计费
- 用户端统一按积分消费
- 不做“每天发多少积分，今日不用就过期”的规则
- 套餐按月发放月度积分，并附带一次性到账的加油包积分

## 六、支付方式

MVP 阶段当前只接入一个支付渠道：

- **支付宝当面付（扫码支付）**

支付流程采用：

1. 用户创建套餐或加油包订单
2. 服务端调用支付宝当面付预下单接口生成二维码
3. 前端展示二维码
4. 用户扫码支付
5. 支付成功后发放对应积分或订阅权益

详细支付方案见 `../支付接入需求文档.md`。

## 七、MVP 边界（套餐与计费）

- 不支持优惠券、折扣码。
- 不支持发票和复杂财务对账。
- 不支持积分转赠。
- 不支持手动拆分账期或补发每日积分。
- 不支持多支付渠道。
- 不支持自动退款。

## 八、验收标准（套餐与计费）

1. 用户可查看套餐和加油包商品。
2. 用户购买套餐后，能同时获得月度积分和附带加油包积分。
3. 用户可单独购买加油包。
4. 用户可查看月度积分和加油包积分余额。
5. AI 消耗积分时，先扣月度积分，再扣加油包积分。
6. 用户可通过支付宝扫码完成套餐或加油包支付。
