# 文档二：金番写作 MVP - 智能对话面板需求文档 V3.4

**模块定位**：右侧固定面板，以“对话”为主视图，支持基于作品、角色、设定和章节信息与 AI 进行持续交流。  
**核心原则**：对话面板负责会话管理、对话、引用和结果落库；编辑器建议由覆盖层承载，不在本面板重复设计第二套入口。

## 一、对话基础能力

| 编号   | 功能               | 描述                                                         | 交互说明                                                           |
| :----- | :----------------- | :----------------------------------------------------------- | :----------------------------------------------------------------- |
| **D1** | 多轮对话界面       | 聊天气泡流，支持流式输出。                                   | 用户消息靠右、黑底白字；AI 消息靠左、白底灰边框、黑字。            |
| **D2** | 会话级历史持久化   | 对话按“会话”保存，而不是只按作品平铺保存。                   | 一个作品下可以有多个会话；每个会话是一条独立对话线程。            |
| **D4** | 引用消息接收       | 从编辑器建议覆盖层点击“发送至对话”后，将引用内容插入为结构化 Mention。 | 展示形态类似 `@章节一:爱人(17行-28行)`，用户可继续补充问题再发送。 |
| **D5** | 发送后清空输入框   | 按 Enter 或点击发送后，输入框清空并保持焦点。                 | Shift+Enter 换行；纯空白消息不可发送；当前引用 Mention 一并清除。 |
| **D5.1** | 模型选择        | 用户发送智能对话前可选择当前启用模型。                       | 仅展示管理端状态为 `active` 的模型。                                |
| **D5.2** | 模型信息展示    | 模型选择器展示模型对外名称、描述、逻辑评分、文笔评分、知识面评分、最大上下文和最大输出。 | 评分范围为 1-5，帮助用户理解模型适用场景。                           |
| **D5.3** | 模型调用 ID     | 用户端只展示模型对外名称，实际请求由服务端使用模型表中的平台调用 ID。 | 平台调用 ID 对用户不可见。                                          |
| **D5.4** | 对话扣费        | AI 对话成功返回后，按真实 OpenAI-compatible usage 计算并扣减积分。 | 用户侧只感知积分消耗，不展示 token 明细。                            |
| **D5.5** | 失败不扣费      | 模型未配置、模型停用、余额预检未通过、请求失败或无有效 AI 输出时不扣积分。 | 需向用户提示失败原因或稍后重试。                                    |
| **D5.6** | 发送前积分预检  | 发送前由服务端按当前模型最大输出和本次上下文估算最高积分消耗。 | 余额不足时禁用发送并提示用户购买套餐或积分权益包。                  |
| **D5.7** | 扣费后余额刷新  | 整段 AI 回复完成并扣费后，前端调用查询接口刷新用户积分余额。 | 对话面板内不展示余额；顶部栏或套餐入口展示最新余额。                |
| **D5.8** | 输入字数显示    | 输入框右下角显示当前输入字数。                               | 不展示单条消息预估扣费、实际扣费或 token 明细。                     |

### 输入、流式与状态规则

- 发送消息使用 Enter；换行使用 Shift+Enter。
- 流式输出过程中，用户可以继续编辑并发送新消息；新消息发送时直接打断当前流式输出，并以新消息开启下一次请求。
- 流式输出过程中提供“停止生成”按钮；用户点击后停止继续接收输出，已显示内容保留在当前 AI 消息中。
- 网络中断导致流式中断时，不额外做恢复或重试；已显示内容保留，并展示通用错误提示。
- 等待 AI 响应时展示打字动画或加载指示。
- 切换会话时展示会话加载状态；向上滚动加载更早消息时展示分页加载指示。
- AI 服务不可用、网络超时、模型报错等场景，消息区域展示通用报错；MVP 不提供失败消息的重试按钮。

### 模型选择与计费说明

- 智能对话模型来源于管理端模型管理模块。
- 模型选择器打开或面板初始化时，必须从服务端远端拉取当前可用模型列表。
- 前端不得写死模型列表；本地缓存只能作为加载中的临时展示，最终以远端返回为准。
- 模型供应商地址、请求协议、API key 管理和调用 ID 规则统一见 `../管理端需求文档/模型管理需求文档.md`。
- 用户只能选择启用模型；停用模型不再出现在模型选择器中。
- 每次成功对话按服务端返回的 usage 扣费，具体计费公式见 `套餐与计费需求文档.md`。
- `reasoning_tokens` 已包含在输出 token 中，不重复计费。
- 发送前必须由服务端按当前模型最大输出和本次上下文估算最高积分消耗；余额不足时不发起模型调用。
- 整段 AI 回复完成后由服务端按真实 usage 扣费，前端再查询最新余额。
- 用户端对话面板不展示剩余积分、单条消息消耗、token 明细或积分流水；简化积分流水在套餐与积分入口查看。

| :----- | :----------------- | :------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **D6** | 会话概念           | 历史记录在产品中统一称为“会话”。                                     | 一个会话对应一个 Agent Session。                                         |
| **D7** | 默认恢复最近会话   | 进入某个作品时，默认打开该作品最近活跃的会话。                       | 若该作品下没有历史会话，则自动创建一个新会话。                           |
| **D8** | 新建会话           | 用户可以主动创建新会话。                                             | 新会话创建后，旧会话保留在历史列表中。                                   |
| **D9** | 历史会话列表       | 面板内提供最近会话列表，用于切换不同历史会话。                       | 按最近活跃时间倒序排列，展示会话标题和最近消息预览。                     |
| **D10** | 会话标题生成      | 会话标题默认由首条用户消息自动生成。                                 | 若首条消息过长，则截断展示；MVP 不支持手动重命名。                       |
| **D11** | 首屏消息加载      | 打开会话时，先加载最近一批消息。                                     | MVP 建议默认加载最近 30 条消息。                                         |
| **D12** | 更早消息按需加载  | 用户向上滚动时，再按页加载更早消息。                                 | 每次分页加载固定数量，例如 20 条，避免首次加载过重。                     |
| **D13** | 历史会话加载范围  | 单个作品下会话数量不做硬性限制，会话列表默认加载近一个月的会话元信息。 | 近一个月内仍可分页加载，避免一次性读取过重。                             |
| **D13.1** | 删除会话        | 用户可以删除单个会话。                                               | 删除需同步处理远端 Agent Session 和本地 `chat_sessions` 映射差异；删除当前会话后切换到最近会话或创建新会话。 |
| **D13.2** | 会话来源        | `source_type` 仅记录会话来源，例如 `manual`、`editor`。               | 从编辑器创建和手动创建的会话在用户侧无行为差异。                         |

## 三、@ 选择器（上下文引用入口）

| 编号    | 功能               | 描述                                                                                               | 交互说明                                                                                                       |
| :------ | :----------------- | :------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------- |
| **D14** | @ 触发浮窗         | 输入 `@` 时弹出浮窗，显示在输入框上方。                                                            | 按 `Esc` 或点击浮窗外区域关闭。                                                                                |
| **D15** | 无关键字时列表规则 | 仅显示：**最近选择（最多 3 项）** + **全部章节（倒序）**，不默认展示全部角色和设定。              | 最近选择中可以同时包含章节、角色和设定；章节列表显示在最近选择下方。                                           |
| **D16** | 有关键字时列表规则 | 根据关键字过滤：**最近选择匹配项置顶** → **匹配的章节（倒序）** → **匹配的角色** → **匹配的设定**。 | 不分区显示，直接按该顺序排列。                                                                                 |
| **D17** | 视觉区分           | 章节、角色、设定在浮窗和消息气泡中使用黑白灰标签样式。                                              | 章节使用浅灰底灰边框，角色使用白底黑边框，设定使用黑底白字；最近选择额外显示浅灰“最近”标记。                   |
| **D18** | 键盘上下导航       | 浮窗打开后，按 `↑` `↓` 移动高亮，按回车选中，按 `Esc` 关闭。                                        | 默认高亮第一项。                                                                                               |
| **D19** | 选中处理           | 选中后，将输入框中 `@` 及后续查询字符替换为结构化 Mention 节点，节点显示为 `@名称`，末尾自动补一个普通空格，并关闭浮窗。 | Mention 节点保存引用对象的 `type`、`id`、`name` 和摘要信息；被选中项加入最近选择列表（最多 3 条，去重，最新置顶）。 |
| **D19.1** | 整体删除        | 已选中的 Mention 节点是一个整体，不允许被拆成普通文本片段。                                             | Backspace 或 Delete 命中节点任意位置时，一次删除整个 `@名称` 节点；选区覆盖节点时先删除整个节点再输入新内容。 |
| **D19.2** | 纯文本发送      | 发送消息时，`message` 只保留用户可见的纯文本，例如 `让 @苏白 和 @苏白 第二次更犹豫`。                    | 不把角色详情、设定详情、章节正文或摘要内容嵌入客户端提交的消息正文；上下文内容由服务端按 ID 获取。             |
| **D19.3** | Mention 锚点发送 | 发送消息时，同时提交有序 `mentions` 数组，记录每一次 Mention 的 `type`、`id`、`label`、`start`、`end`。 | `mentions` 表达展示锚点和出现顺序，允许同一 `type` + `id` 出现多次；文本中的 `@名称` 解析仅作为兼容兜底。     |
| **D19.4** | 上下文引用去重  | 服务端根据 `mentions` 和兼容引用派生去重后的上下文引用集合。                                           | 同一 `type` + `id` 只取一份内容，用于 Agent 工具调用或上下文注入；去重不得改写消息正文，也不得丢失 `mentions` 的出现顺序。 |
| **D20** | @ 内容高亮         | 已发送消息气泡中，`@角色名`、`@设定名` 和 `@章节名` 以标签样式高亮显示。                             | 高亮依据消息携带的有序 `mentions` 锚点；文本复制仍保持为普通 `@名称`。                                        |
| **D20.1** | 新会话默认章节  | 新建会话时，若存在当前章节，输入框默认插入当前章节 Mention。                                           | 用户可手动删除该 Mention；当前章节不再作为不可见默认上下文自动注入。                                          |

## 四、自动上下文注入与 Agent 策略

| 编号    | 功能               | 描述                                                                                   | 交互说明                                                                 |
| :------ | :----------------- | :------------------------------------------------------------------------------------- | :----------------------------------------------------------------------- |
| **D21** | 上下文组装         | 发送消息时，服务端优先根据结构化 `mentions` 的 `type` + `id` 获取角色、设定和章节上下文，并读取当前作品信息。 | 当前章节只有在输入框中保留对应 Mention 时才作为章节上下文注入；Agent 可通过后端工具按 ID 获取所需内容。 |
| **D22** | 历史上下文注入     | Agent 默认带入最近几轮历史对话，而不是全量历史。                                        | MVP 建议带入最近 3 轮运行记录或最近 20 条消息，以平衡效果与成本。         |
| **D23** | 长会话优化策略     | 当单个会话过长时，可启用会话摘要机制减少上下文长度。                                    | MVP 第一版可以先不默认开启，等会话长度和成本上来后再打开。               |
| **D24** | 注入格式           | 结构化注入作品信息、重点要求、禁忌要求、角色信息、设定详情、章节提要和最近历史对话。     | 重点要求和禁忌要求必须进入系统提示词；同一会话中作品信息变化后，后续请求的系统提示词同步变化。 |

## 五、Agent 工具

Agent 配备以下工具，用于按需读取、创建/更新和删除作品相关实体。工具调用由 Agent 根据对话上下文自主决策，用户无需手动触发。

工具调用结果需要在消息流中对用户可见：

- 创建或更新成功时，在对话气泡中展示类似“已创建角色 XXX”“已更新设定 XXX”的反馈。
- 工具调用失败时，按工具结果正常展示失败信息，不做静默吞掉。
- 若 Agent 处理时引用的角色、设定或章节已被删除，对应工具直接返回错误，并在消息流中展示。

Agent 工具在不同加载状态下都需要保持可用和可感知：

- 流式输出过程中，Agent 工具调用可穿插发生；工具执行中、成功和失败状态需要能随着流式消息同步展示。
- 本地已有会话消息或本地缓存状态恢复时，历史工具调用结果需要正常展示，不丢失“已创建/已更新/失败”等反馈。
- 远端加载会话、消息和 Agent Session 时，工具调用记录需要随消息一并加载；加载期间展示加载状态，加载失败时展示通用错误。
- 本地状态与远端状态不一致时，以远端返回的工具调用结果和实体状态为准。

## 六、MVP 边界（对话面板部分）

- 不设置独立的”建议”标签页，建议能力完全由编辑器覆盖层触发。
- 不支持图片上传、对话导出、复杂富文本排版。
- 不做快捷指令按钮。
- 不支持历史会话搜索。
- 不支持会话重命名、置顶、归档。
- 不支持跨会话上下文搜索。
- 不设置人工确认的”保存为角色/设定”等操作按钮，数据写入由 Agent 工具直接完成。
- 不在对话面板展示剩余积分、单条消息扣费、token 明细和积分流水。

## 七、验收标准（对话面板部分）

1. 对话可正常发送和接收，消息气泡样式区分用户与 AI。
2. 进入作品时，可自动恢复最近活跃的会话；若无会话则自动创建新会话。
3. 用户可查看最近会话列表，并在不同会话间切换。
4. 打开会话时先加载最近消息，更早消息支持滚动分页加载。
5. 输入 `@` 后弹出浮窗；无关键字时仅显示最近选择和章节列表。
6. 输入 `@+关键字` 后，可正确过滤并显示匹配的章节、角色和设定。
7. 键盘上下导航、回车选中和最近选择更新行为正确。
8. 选择 `@` 引用后，输入框中的 `@名称` 作为整体 Mention 节点存在，Backspace/Delete 一次删除整个节点。
9. 同一条输入中多次提及同一对象时，`message` 保持纯文本并按输入顺序展示每一次 `@名称`，`mentions` 保留每一次出现，服务端派生的上下文引用对同一 `type` + `id` 只取一份内容。
10. 发送后，消息中的 `@` 内容按章节、角色和设定分别高亮显示。
11. AI 回复能够体现当前作品信息、被引用角色、设定、章节和最近历史对话。
12. Agent 可通过工具读取作品、章节、角色和设定的完整信息。
13. Agent 可在对话中自主调用 create_or_update_character 等工具创建或更新实体。
14. Agent 可通过工具删除角色和设定。
15. Agent 可通过工具更新章节提要和作品信息字段。
16. 用户只能选择管理端启用的模型，并能看到模型名称、描述、三项评分、最大上下文和最大输出。
17. 对话发送前服务端按当前模型最大输出和本次上下文估算最高积分消耗；余额不足时不发起模型调用。
18. 对话成功返回后按真实 usage 扣费；模型未配置、模型停用、余额预检未通过、请求失败或无有效输出时不扣费。
19. 余额不足时，发送按钮不可用并提示用户购买套餐或积分权益包。
20. 新建会话时默认插入当前章节 Mention；用户删除该 Mention 后，当前章节不再注入本次请求上下文。
21. 系统提示词必须包含作品重点要求和禁忌要求；作品信息变更后，后续请求使用最新约束。
22. 流式输出可被“停止生成”按钮停止；用户发送新消息时会打断当前流式输出。
23. AI 响应失败或流式网络中断时展示通用报错，MVP 不提供重试按钮。
24. Agent 工具执行成功或失败结果可在消息流中看到。
