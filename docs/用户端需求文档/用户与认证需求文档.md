# 文档七：金番写作 MVP - 用户与认证需求文档 V1.0

**模块定位**：负责用户注册、登录、退出、账户信息展示以及基础权限控制。  
**核心原则**：认证流程保持最小可用，用户与管理员共用一套账号体系，通过角色区分权限。

## 一、账户基础能力

| 编号   | 功能         | 描述                                                         | 交互说明                                                                 |
| :----- | :----------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **A1** | 用户注册     | 支持通过邮箱和密码创建账户。                                 | 注册成功后进入用户端工作台。                                             |
| **A2** | 用户登录     | 支持通过邮箱和密码登录。                                     | 登录成功后跳转到作品列表页。                                             |
| **A3** | 用户退出     | 当前登录用户可主动退出。                                     | 退出后清理登录态并返回登录页。                                           |
| **A4** | 账户信息查看 | 用户可查看自己的基础信息。                                   | MVP 先展示邮箱、昵称、账户状态、当前套餐、当前积分。                     |
| **A5** | 昵称修改     | 用户可修改昵称。                                             | 昵称修改成功后立即生效。                                                 |

## 二、角色与权限

| 编号   | 功能           | 描述                                                         | 交互说明                                                                 |
| :----- | :------------- | :----------------------------------------------------------- | :----------------------------------------------------------------------- |
| **A6** | 统一账号体系   | 普通用户和管理员共用 `users` 表。                            | 通过 `role` 区分 `user` 和 `admin`。                                     |
| **A7** | 用户端权限     | `user` 角色只能访问自己的作品、角色、设定、章节和会话。      | 任何跨用户访问都应被拒绝。                                               |
| **A8** | 管理端权限     | `admin` 角色可访问管理端后台。                               | 管理端入口和后台接口需做角色校验。                                       |
| **A9** | 账户状态控制   | 账户支持 `active`、`disabled` 两种基础状态。                 | 被禁用用户不能登录。                                                     |

## 三、MVP 边界（用户与认证）

- 不支持手机号注册登录。
- 不支持第三方登录。
- 不支持复杂权限模型。
- 不支持找回密码和邮件验证流程。

## 四、验收标准（用户与认证）

1. 用户可通过邮箱和密码注册、登录、退出。
2. 用户可查看并修改自己的昵称。
3. 系统能够根据 `role` 区分用户端和管理端访问权限。
4. 被禁用账户无法登录。
