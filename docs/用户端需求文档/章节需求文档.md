# 文档六：金番写作 MVP - 章节需求文档 V1.0

**模块定位**：章节模块用于管理作品下的章节列表和每章正文内容。  
**核心原则**：章节结构保持最小化，每章只保留标题、顺序、正文和章节提要四项核心信息。

## 一、章节基础能力

| 编号   | 功能       | 描述                                                     | 交互说明                                                                 |
| :----- | :--------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **C1** | 章节列表   | 展示当前作品下全部章节。                                 | 每项至少显示章节顺序、章节标题和章节提要预览。                           |
| **C2** | 新建章节   | 用户可在当前作品下创建章节。                             | 默认生成占位标题，创建后立即进入编辑状态。                               |
| **C3** | 编辑章节   | 用户可修改章节标题、正文和章节提要。                     | 修改后内容自动保存或手动保存。                                           |
| **C4** | 删除章节   | 用户可删除章节。                                         | 删除前需二次确认；删除当前章节后自动切换到相邻章节或空状态。             |
| **C5** | 调整顺序   | 用户可调整章节顺序。                                     | 调整后同步刷新列表顺序和章节编号。                                       |
| **C6** | 切换章节   | 点击章节后加载该章节正文和章节提要。                     | 编辑器与对话默认上下文同步切换到当前章节。                               |

## 二、章节字段定义

| 字段名         | 必填 | 描述                                                     |
| :------------- | :--- | :------------------------------------------------------- |
| **章节顺序**   | 是   | 章节在当前作品中的排序位置。                             |
| **章节标题**   | 是   | 当前章节名称。                                           |
| **章节正文**   | 是   | 当前章节的正文内容。                                     |
| **章节提要**   | 否   | 对当前章节内容的简要说明，用于快速回顾和 AI 上下文注入。 |

## 三、保存与展示

| 编号   | 功能         | 描述                                                     | 交互说明                                                                 |
| :----- | :----------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **C7** | 自动保存     | 章节正文和章节提要支持自动保存。                         | 可采用节流或停止输入后自动保存策略。                                     |
| **C8** | 保存状态提示 | 页面需明确展示当前保存状态。                             | 至少区分“保存中”“已保存”“保存失败”三种状态。                           |
| **C9** | 字数统计     | 编辑器中显示当前章节字数。                               | 字数根据章节正文实时计算。                                               |
| **C10** | 提要预览    | 章节列表中展示章节提要摘要。                             | 方便快速浏览每章大致内容。                                               |

## 四、与 AI 的联动能力

| 编号   | 功能             | 描述                                                     | 交互说明                                                                 |
| :----- | :--------------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **C11** | 当前章节上下文  | 当前章节默认作为对话面板的重要上下文之一。               | 即使用户未显式 `@章节名`，系统也可默认带上当前章节内容和章节提要。       |
| **C12** | 对话引用        | 章节可在对话面板中通过 `@章节名` 被引用。                | 引用后，后端自动把该章节正文摘要和章节提要注入上下文。                   |
| **C13** | 从对话更新提要  | 对话中 AI 生成的章节摘要可一键写入章节提要。             | 用户选择目标章节后写入章节提要字段。                                     |
| **C14** | 编辑器分析范围  | 编辑器 AI 分析默认只作用于当前章节正文。                 | 切换章节后，高亮和建议结果需重新加载。                                   |

## 五、MVP 边界（章节部分）

- 不支持多章同时编辑。
- 不支持章节版本历史和回滚。
- 不支持批量导入导出章节。
- 不做独立的章节大纲模块。

## 六、验收标准（章节部分）

1. 用户可创建、编辑、删除和排序章节。
2. 章节可保存顺序、标题、正文和章节提要。
3. 切换章节后，编辑器正确加载对应章节内容。
4. 章节内容支持保存状态提示和自动保存。
5. 对话面板中可通过 `@章节名` 找到并引用章节。
6. 对话中生成的章节摘要可写入章节提要。
