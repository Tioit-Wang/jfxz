# 文档一：金番写作 MVP - 编辑器需求文档 V1.1

**模块定位**：基于 TipTap 的纯文本写作区域，配合 AI 高亮标记与建议覆盖层，帮助作者在不离开当前页面的情况下发现问题并处理问题。  
**核心原则**：编辑过程尽量简单，辅助信息只在需要时出现。

**技术约束**：正文编辑器必须使用 TipTap 实现，但用户能力与数据存储仍保持纯文本；不得提供富文本工具栏、图片、表格或复杂排版入口。

## 一、编辑器基础能力

| 编号   | 功能           | 描述                                         | 交互说明                                                                          |
| :----- | :------------- | :------------------------------------------- | :-------------------------------------------------------------------------------- |
| **E1** | 纯文本编辑     | 中央大面积写作区域，仅支持纯文本输入与粘贴。 | 无富文本工具栏，内容可直接复制到目标发布平台或其他系统。                         |
| **E2** | 字数实时统计   | 编辑器底部常驻状态栏，显示两栏信息。         | 左侧显示“本章字数: XXXX”；右侧显示“今日字数: XXXX”，输入时实时更新。             |
| **E3** | 普通选中无干扰 | 用户划词、复制、粘贴时不触发额外工具。       | 系统不拦截文字选中行为，不弹出快捷浮层。                                          |
| **E3-1** | 自动保存       | 停止输入后自动保存当前章节标题、提要和正文。 | 切换章节、删除章节、AI 检测、采纳建议前均需先保存当前章节；保存失败时阻止对应切换或操作并提示。 |

## 二、AI 高亮标记与气泡

| 编号   | 功能             | 描述                                                                      | 交互说明                                                                                               |
| :----- | :--------------- | :------------------------------------------------------------------------ | :----------------------------------------------------------------------------------------------------- |
| **E4** | AI 自动高亮      | AI 分析当前章节后，对需要关注的段落添加淡黄色背景（`#FEF9E7` 或类似色）。 | 高亮是系统标记，不是用户格式。切换章节或重新分析后，高亮结果同步更新。                                 |
| **E5** | 悬停气泡         | 鼠标移入高亮段落时，在段落附近显示提示气泡。                              | 例如：“此处角色表现与角色资料不一致。” 或 “此处内容与当前设定不一致。” 鼠标移开后气泡消失。            |
| **E6** | 点击高亮触发建议 | 鼠标点击高亮段落时，触发右侧对话面板上方的建议覆盖层。                    | 单击即触发。覆盖层出现后，当前高亮段落可进一步强调显示；关闭后恢复原状态。                             |
| **E7** | 普通正文点击     | 点击未被高亮的正文区域时，不触发建议覆盖层。                              | 仅执行正常的光标定位和编辑行为。                                                                       |
| **E7-1** | 编辑后清空旧检测 | 用户修改正文后，清空当前章节旧高亮与旧建议。                              | 避免旧检测结果对应不到新正文。                                                                         |
| **E7-2** | 检查模型配置   | AI 检查使用管理端系统配置指定的检查模型。                                  | 用户端不可在编辑器检查入口切换模型。                                                                   |

## 三、建议覆盖层（在对话面板上方呈现）

| 编号    | 功能             | 描述                                                                                                                                 | 交互说明                                                                   |
| :------ | :--------------- | :----------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------- |
| **E8**  | 触发与展示位置   | 用户点击高亮段落后，右侧对话面板的消息列表区域显示建议覆盖层。                                                                       | 覆盖层与对话面板等宽，覆盖原消息区内容，对话历史不丢失。                   |
| **E9**  | 覆盖层内容       | 展示当前选中段落的完整建议列表。每张建议卡片包含：原文引用、问题描述、一个或多个修改方案。                                          | 覆盖层内部可滚动；当方案较多时，允许在覆盖层内浏览全部建议。               |
| **E10** | 操作按钮         | 每张卡片底部包含两个按钮：“采纳替换”与“发送至对话”。                                                                                | 采纳后覆盖层自动关闭；发送至对话后覆盖层关闭，对话输入框自动填入引用内容。 |
| **E11** | 关闭覆盖层       | 覆盖层右上角提供明显的关闭按钮。                                                                                                     | 点击关闭后，覆盖层消失，对话面板恢复显示，对话历史保持不变。               |
| **E12** | 连续点击不同高亮 | 在编辑器中点击其他高亮段落时，覆盖层内容即时刷新为对应段落的建议。                                                                   | 无需先手动关闭再重新打开。                                                 |

## 四、MVP 边界（编辑器部分）

- 不提供富文本格式工具栏。
- 不支持图片、表格等复杂内容元素。
- 不支持同时编辑多个章节。
- AI 分析触发方式在 MVP 阶段可简化为手动触发。
- AI 检测 MVP 仅检查错别字、错误标点符号、明显病句或不通顺表达。
- AI 检测前先保存当前章节，并由服务端按检查模型最大输出和本次章节上下文估算最高积分消耗。余额预检通过后才发起模型调用；检测成功后按真实 OpenAI-compatible usage 扣减积分。余额预检未通过、未配置模型、模型停用、上游失败、模型返回结构解析失败或无有效结果时不扣积分，并向用户提示。
- AI 检测模型由管理端系统配置项指定，例如 `ai.editor_check.model_id`。用户端不提供前端切换入口。

## 五、验收标准（编辑器部分）

1. 编辑器为纯文本模式，无格式工具栏，可正常输入、粘贴、复制。
2. 字数实时更新，“本章字数”和“今日字数”显示正确。
3. AI 分析后，高亮段落可见淡黄色背景。
4. 悬停高亮段落时显示气泡，移开后消失。
5. 点击高亮段落时，右侧出现建议覆盖层，且包含原文引用和修改方案。
6. 点击“采纳替换”后，编辑器对应段落替换成功，覆盖层关闭。
7. 点击“发送至对话”后，覆盖层关闭，对话输入框出现对应引用内容。
8. 点击关闭按钮后，覆盖层正常关闭，对话历史内容不丢失。
9. 普通划词和复制行为不会触发任何额外弹窗。
10. 自动保存、提要弹框保存、章节切换前保存、AI 检测前保存和采纳建议后保存均能写入真实章节数据。
11. AI 检测无问题时给出“未发现明显问题”类提示，不打开空建议覆盖层。
12. AI 检测使用管理端配置的检查模型，用户端不可切换模型。
13. AI 检测前服务端按检查模型最大输出和本次章节上下文估算最高积分消耗；余额不足时不发起模型调用。
14. AI 检测成功后按真实 usage 扣费；余额预检未通过、解析失败、模型未配置、模型停用、上游失败或无有效结果时给出明确提示，且不扣积分。
