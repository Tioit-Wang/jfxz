# 文档四：金番写作 MVP - 角色需求文档 V1.0

**模块定位**：角色是独立于通用设定之外的人物信息模块，用于管理作品中的所有人物资料。  
**核心原则**：角色单独管理，避免与地点、装备、规则等非人物设定混在一起。

## 一、角色基础能力

| 编号   | 功能       | 描述                                                     | 交互说明                                                                 |
| :----- | :--------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **R1** | 角色列表   | 展示当前作品下的全部角色。                               | 每项至少显示角色名称、角色简介、最近更新时间。                           |
| **R2** | 新建角色   | 用户可在当前作品下创建角色。                             | 点击“新建角色”后打开表单，保存后进入角色详情。                           |
| **R3** | 编辑角色   | 用户可修改已有角色信息。                                 | 修改后保存，列表与详情同步更新。                                         |
| **R4** | 删除角色   | 用户可删除不再使用的角色。                               | 删除前需二次确认。                                                       |
| **R5** | 关键字搜索 | 支持按角色名称和简介搜索。                               | 搜索结果实时刷新。                                                       |

## 二、角色字段定义

MVP 阶段角色不做复杂强结构，先保留够用的核心字段。

| 字段名       | 必填 | 描述                                                     |
| :----------- | :--- | :------------------------------------------------------- |
| **角色名称** | 是   | 角色主名称。                                             |
| **角色简介** | 是   | 用于快速说明这个角色是谁。                               |
| **角色详情** | 否   | 用于补充角色的更完整信息，例如经历、性格、能力、关系等。 |

## 三、角色详情展示

| 编号   | 功能         | 描述                                                     | 交互说明                                                                 |
| :----- | :----------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **R6** | 详情查看     | 点击角色后进入详情视图。                                 | 展示角色名称、简介、详情和更新时间。                                     |
| **R7** | 快速复制     | 支持复制角色名称、简介或完整详情。                       | 方便作者在编辑器或其他场景中手动使用。                                   |
| **R8** | 当前作品绑定 | 所有角色都归属当前作品，不允许跨作品混用。               | 作品切换后，角色列表同步切换。                                           |

## 四、与 AI 的联动能力

| 编号   | 功能           | 描述                                                     | 交互说明                                                                 |
| :----- | :------------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **R9** | 对话引用       | 角色可在对话面板中通过 `@角色名` 被引用。                | 引用后，后端自动把该角色信息注入对话上下文。                             |
| **R10** | 从对话保存    | MVP 阶段对话中 AI 生成的角色草稿由 Agent 工具直接写入角色模块。 | 不设置人工确认按钮；创建或更新结果需在对话消息流中反馈给用户。           |
| **R11** | 编辑器联动    | 编辑器分析时可引用角色信息判断正文是否与角色资料一致。   | 例如角色行为、说话方式与角色资料不一致时进行提示。                       |

## 五、MVP 边界（角色部分）

- 不做复杂人物关系图。
- 不做角色版本历史和回滚。
- 不做角色自动抽取和自动去重。

## 六、验收标准（角色部分）

1. 用户可创建、编辑、删除角色。
2. 角色可保存名称、简介和详情。
3. 角色列表支持关键字搜索。
4. 对话面板中可通过 `@角色名` 找到并引用角色。
5. 对话中生成的角色草稿可由 Agent 工具直接写入角色模块，并在消息流中展示结果。
6. 编辑器分析可读取角色信息进行一致性判断。
