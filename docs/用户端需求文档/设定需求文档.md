# 文档五：金番写作 MVP - 设定需求文档 V1.0

**模块定位**：设定模块用于管理除角色外的非人物信息，例如地点、装备、属性、规则和其他作品设定。  
**核心原则**：角色单独管理，其余长期有效的作品信息统一归入设定模块。

## 一、设定基础能力

| 编号   | 功能       | 描述                                                     | 交互说明                                                                 |
| :----- | :--------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **S1** | 设定列表   | 展示当前作品下全部设定。                                 | 每项至少显示设定名称、设定类型、设定简介、最近更新时间。                 |
| **S2** | 新建设定   | 用户可在当前作品下创建设定。                             | 点击“新建设定”后打开表单，保存后进入详情。                               |
| **S3** | 编辑设定   | 用户可修改已有设定信息。                                 | 保存后列表和详情同步刷新。                                               |
| **S4** | 删除设定   | 用户可删除不再需要的设定。                               | 删除前需二次确认。                                                       |
| **S5** | 搜索与筛选 | 支持按设定名称搜索，并按设定类型筛选。                   | 搜索和筛选可同时生效。                                                   |

## 二、设定字段定义

| 字段名       | 必填 | 描述                                                                 |
| :----------- | :--- | :------------------------------------------------------------------- |
| **设定类型** | 是   | 用于区分设定类别，例如地点、装备、属性、规则、组织、其他。           |
| **设定名称** | 是   | 设定主名称。                                                         |
| **设定简介** | 是   | 用于快速说明这条设定是什么。                                         |
| **设定详情** | 否   | 用于补充更完整内容，例如来源、作用、限制、使用方式等。               |

## 三、设定详情展示

| 编号   | 功能         | 描述                                                     | 交互说明                                                                 |
| :----- | :----------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **S6** | 详情查看     | 点击设定后进入详情视图。                                 | 展示设定类型、名称、简介、详情和更新时间。                               |
| **S7** | 快速复制     | 支持复制设定名称、简介或完整详情。                       | 方便作者在编辑器或其他场景中手动使用。                                   |
| **S8** | 当前作品绑定 | 所有设定都归属当前作品，不允许跨作品混用。               | 作品切换后，设定列表同步切换。                                           |

## 四、与 AI 的联动能力

| 编号   | 功能           | 描述                                                     | 交互说明                                                                 |
| :----- | :------------- | :------------------------------------------------------- | :----------------------------------------------------------------------- |
| **S9** | 对话引用       | 设定可在对话面板中通过 `@设定名` 被引用。                | 引用后，后端自动把该设定内容注入对话上下文。                             |
| **S10** | 从对话保存    | MVP 阶段对话中 AI 生成的设定草稿由 Agent 工具直接写入设定模块。 | 不设置人工确认按钮；创建或更新结果需在对话消息流中反馈给用户。           |
| **S11** | 编辑器联动    | 编辑器分析时可引用设定信息判断正文是否与当前设定一致。   | 例如地点、装备、属性或规则前后不一致时给出提示。                         |

## 五、MVP 边界（设定部分）

- 不做复杂关系图。
- 不做版本历史和回滚。
- 不做自动抽取和自动去重。

## 六、验收标准（设定部分）

1. 用户可创建、编辑、删除设定。
2. 设定可保存类型、名称、简介和详情。
3. 设定列表支持关键字搜索和类型筛选。
4. 对话面板中可通过 `@设定名` 找到并引用设定。
5. 对话中生成的设定草稿可由 Agent 工具直接写入设定模块，并在消息流中展示结果。
6. 编辑器分析可读取设定信息进行一致性判断。
