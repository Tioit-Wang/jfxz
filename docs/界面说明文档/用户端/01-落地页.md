# 用户端 - 落地页界面说明

## 一、界面定位

落地页是未登录用户进入产品后的第一屏，用于介绍产品价值，并引导用户开始写作。

对应路由：`/`

## 二、界面长什么样

页面采用常规官网落地页结构。

```text
┌──────────────────────────────┐
│ 顶部导航：Logo / 产品介绍 / 价格 / 开始写作 │
├──────────────────────────────┤
│ 首屏 Hero：一句话卖点 + 简短说明 + 开始写作按钮 │
├──────────────────────────────┤
│ 核心能力区：作品管理 / 角色设定 / AI 对话 / AI 分析 │
├──────────────────────────────┤
│ 使用流程区：创建作品 → 写章节 → 问 AI → 采纳建议 │
├──────────────────────────────┤
│ 套餐价格入口：展示基础套餐和加油包入口 │
├──────────────────────────────┤
│ 底部：备案/版权/联系方式，可后续补充 │
└──────────────────────────────┘
```

## 三、需要实现的功能

- 展示产品名称和核心定位。
- 展示产品主要能力。
- 展示基础使用流程。
- 提供“开始写作”按钮。
- 提供“查看价格”入口。
- 已登录用户点击“开始写作”进入 `/books`。
- 未登录用户点击“开始写作”弹出登录注册模态框。

## 四、交互说明

- 点击“开始写作”：
  - 已登录：跳转 `/books`。
  - 未登录：打开登录注册模态框。
- 点击“查看价格”：
  - 如果落地页内有价格区，则滚动到价格区。
  - 如果后续有独立价格弹窗，也可以打开套餐与积分模态框。
- 点击顶部 Logo：回到页面顶部。

## 五、状态说明

- 未登录状态：显示“开始写作”。
- 已登录状态：显示“进入工作台”。
- 加载状态：按钮可显示 loading，避免重复点击。

## 六、MVP 边界

- 不做复杂动效。
- 不做多语言。
- 不做复杂营销页组件。
- 不做独立 SEO 内容运营系统。
