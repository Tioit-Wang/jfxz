# 用户端 - 登录注册模态框界面说明

## 一、界面定位

登录注册模态框用于完成用户身份确认。用户端不设置独立登录页和注册页，登录与注册统一在该模态框中完成。

## 二、界面长什么样

这是一个居中的大模态框，覆盖当前页面。

```text
┌──────────────────────┐
│ 使用邮箱继续          │
│ 简短说明：登录或创建账号 │
│                      │
│ 邮箱输入框             │
│ 验证码/密码输入区域，可按实现决定 │
│                      │
│ 继续按钮               │
│                      │
│ 说明：未注册邮箱将自动创建账号 │
└──────────────────────┘
```

## 三、需要实现的功能

- 输入邮箱。
- 提交邮箱。
- 判断邮箱是否已有账号。
- 已有账号则登录。
- 没有账号则自动注册并登录。
- 登录成功后关闭模态框。
- 登录成功后继续执行用户原本想做的操作。

## 四、交互说明

触发场景：

- 落地页点击“开始写作”。
- 未登录访问 `/books`。
- 未登录访问 `/books/:bookId`。
- 未登录用户点击创建作品。
- 未登录用户保存章节。
- 未登录用户发送 AI 对话。
- 未登录用户购买套餐。

提交逻辑：

1. 用户输入邮箱。
2. 点击“继续”。
3. 系统校验邮箱格式。
4. 系统查询邮箱是否存在。
5. 如果存在，完成登录。
6. 如果不存在，自动创建用户并完成登录。
7. 关闭模态框。
8. 继续原操作或跳转目标页面。

## 五、状态说明

- 默认状态：展示邮箱输入框和继续按钮。
- 输入错误：提示邮箱格式不正确。
- 提交中：按钮显示 loading，不允许重复提交。
- 登录成功：关闭模态框。
- 登录失败：展示错误提示。
- 账号禁用：提示账号不可用。

## 六、MVP 边界

- 不做独立注册页。
- 不做手机号登录。
- 不做第三方登录。
- 不做找回密码。
- 不做复杂账户安全流程。
