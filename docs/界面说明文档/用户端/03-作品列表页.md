# 用户端 - 作品列表页界面说明

## 一、界面定位

作品列表页展示当前用户创建的全部作品，是用户进入具体创作工作台之前的入口。

对应路由：`/books`

## 二、界面长什么样

```text
┌──────────────────────────────┐
│ 顶部栏：Logo / 账户 / 积分 / 新建作品 │
├──────────────────────────────┤
│ 页面标题：我的作品              │
│ 新建作品按钮                   │
├──────────────────────────────┤
│ 作品卡片列表                   │
│ ┌────────┐ ┌────────┐ ┌────────┐ │
│ │作品名   │ │作品名   │ │作品名   │ │
│ │短简介   │ │短简介   │ │短简介   │ │
│ │更新时间 │ │更新时间 │ │更新时间 │ │
│ └────────┘ └────────┘ └────────┘ │
└──────────────────────────────┘
```

## 三、需要实现的功能

- 展示作品列表。
- 每个作品显示作品名称、短简介、最近更新时间。
- 新建作品。
- 进入作品工作台。
- 删除作品。
- 空状态展示。
- 未登录访问时弹出登录注册模态框。

## 四、交互说明

- 点击作品卡片：进入 `/books/:bookId`。
- 点击“新建作品”：打开新建作品模态框。
- 新建作品保存成功：进入该作品工作台。
- 点击删除：弹出二次确认。
- 删除成功：刷新作品列表。
- 未登录访问：弹出登录注册模态框，登录成功后加载作品列表。

## 五、状态说明

- 加载中：展示骨架屏或加载提示。
- 空列表：展示“还没有作品，创建第一本作品”。
- 删除中：删除按钮 loading。
- 删除失败：提示错误。

## 六、MVP 边界

- 不做作品分组。
- 不做作品排序配置。
- 不做作品封面。
- 不做作品间复制。
