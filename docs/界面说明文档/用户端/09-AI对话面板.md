# 用户端 - AI 对话面板界面说明

## 一、界面定位

AI 对话面板位于作品工作台右侧，用于基于当前作品、章节、角色和设定与 AI 持续对话。

## 二、界面长什么样

整体视觉使用黑白简约色调：用户消息为黑底白字，AI 消息为白底黑字和灰色边框；章节、角色、设定引用通过黑白灰标签的底色、边框和字重区分，不使用彩色主色。

```text
┌──────────────┐
│ 当前会话标题  │
│ 新建 / 历史   │
├──────────────┤
│ AI 消息       │
│ 用户消息      │
│ AI 消息       │
├──────────────┤
│ 快捷指令按钮  │
│ 引用块，可选  │
│ 输入框 + 发送 │
└──────────────┘
```

## 三、需要实现的功能

- 展示当前会话消息。
- 发送用户消息。
- 接收 AI 流式回复。
- 新建会话。
- 打开会话历史。
- 快捷指令。
- `@` 引用章节、角色、设定。
- 展示引用块。
- 消息中的 `@` 内容高亮。
- AI 回复下方出现可落地操作，例如保存为角色、保存为设定、更新章节提要。

## 四、交互说明

- 进入作品工作台后默认恢复最近活跃会话。
- 没有会话时自动创建新会话。
- 点击新建会话后开始新的对话线程。
- 输入 `@` 弹出引用选择浮窗。
- 选择引用项后，输入框中插入结构化 `@名称` Mention 节点。
- Mention 节点是一个整体，不能在中间拆开编辑；Backspace 或 Delete 命中节点时一次删除整个 `@名称`。
- 发送时输入框输出普通文本，同时把 Mention 节点携带的章节、角色、设定引用作为结构化 references 一并提交。
- 点击发送后清空输入框并保持焦点。
- 从 AI 建议覆盖层点击“发送至对话”后，输入框上方出现引用块。

## 五、状态说明

- 会话加载中。
- 消息加载中。
- AI 回复中。
- 发送失败。
- 无历史会话。
- 积分不足。

## 六、MVP 边界

- 不做图片上传。
- 不做对话导出。
- 不做会话重命名。
- 不做会话置顶和归档。
