# 用户端 - AI 建议覆盖层界面说明

## 一、界面定位

AI 建议覆盖层用于展示编辑器 AI 分析后针对某个高亮段落的具体修改建议。

## 二、界面长什么样

覆盖层出现在右侧 AI 对话面板的消息区域上方，不覆盖整个页面。

```text
┌──────────────┐
│ 建议详情   X │
├──────────────┤
│ 原文引用      │
│ 问题描述      │
│ 修改方案 A    │
│ 修改方案 B    │
│ 采纳替换 / 发送至对话 │
└──────────────┘
```

## 三、需要实现的功能

- 展示当前高亮段落原文。
- 展示问题描述。
- 展示一个或多个修改方案。
- 支持展示同一轮检测中的多条建议。
- 采纳替换。
- 发送至对话。
- 关闭覆盖层。
- 连续点击不同高亮时刷新内容。

## 四、交互说明

- 点击编辑器高亮段落：打开覆盖层。
- 点击其他高亮段落：覆盖层内容切换为新段落建议。
- 点击“采纳替换”：用建议内容替换原文引用对应内容，关闭覆盖层，并触发章节保存。
- 点击“发送至对话”：关闭覆盖层，在 AI 对话输入框上方生成引用块，引用块包含原文、问题描述和建议修改。
- 点击关闭：覆盖层消失，对话历史恢复显示。
- 正文被继续编辑后，旧建议覆盖层关闭，旧高亮清空。

## 五、状态说明

- 建议加载中。
- 无建议。
- 替换中。
- 替换失败。
- 已发送至对话。

## 六、MVP 边界

- 不做独立建议页。
- 不做建议历史列表。
- 不做多段落批量采纳。
