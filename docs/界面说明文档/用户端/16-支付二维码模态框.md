# 用户端 - 支付二维码模态框界面说明

## 一、界面定位

支付二维码模态框用于展示支付宝当面付二维码，并反馈支付状态。

## 二、界面长什么样

```text
┌──────────────────────┐
│ 支付订单              │
│ 商品名称 / 金额        │
├──────────────────────┤
│ 支付二维码             │
├──────────────────────┤
│ 状态：等待支付          │
│ 取消 / 我已支付         │
└──────────────────────┘
```

## 三、需要实现的功能

- 展示订单商品名称。
- 展示订单金额。
- 展示支付宝支付二维码。
- 轮询支付状态。
- 支付成功后提示成功。
- 支付成功后刷新积分或订阅权益。

## 四、交互说明

- 创建订单成功后打开。
- 用户扫码支付。
- 前端定时查询支付状态。
- 支付成功后展示成功状态，并允许关闭。
- 关闭后返回套餐与积分模态框。

## 五、状态说明

- 二维码加载中。
- 等待支付。
- 支付成功。
- 支付失败。
- 支付超时。
- 查询状态失败。

## 六、MVP 边界

- 只支持支付宝当面付。
- 不支持多支付渠道。
- 不支持自动退款。
