# 管理端 - 订单详情抽屉界面说明

## 一、界面定位

订单详情抽屉用于查看单个订单的基础信息、支付信息和积分发放来源。

## 二、界面长什么样

```text
┌──────────────────────┐
│ 订单详情              │
├──────────────────────┤
│ 订单编号              │
│ 用户                  │
│ 商品类型              │
│ 商品名称              │
│ 金额                  │
│ 状态                  │
│ 创建时间              │
│ 支付时间              │
├──────────────────────┤
│ 积分发放记录           │
│ 支付记录               │
└──────────────────────┘
```

## 三、需要实现的功能

- 查看订单基础信息。
- 查看支付状态。
- 查看积分发放来源。
- 查看关联支付记录。

## 四、交互说明

- 从订单管理页点击详情打开。
- 点击关闭返回订单列表。
- 支付记录区域可展开查看更多信息。

## 五、状态说明

- 加载中。
- 加载失败。
- 无支付记录。
- 无积分发放记录。

## 六、MVP 边界

- 不做订单编辑。
- 不做退款操作。
- 不做人工补发积分。
