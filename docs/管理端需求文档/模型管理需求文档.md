# 金番写作 MVP - 模型管理需求文档

## 一、目标

管理端模型管理模块用于维护可用于用户端智能对话和编辑器 AI 检查的模型目录。

模型调用相关统一口径以本文档为准，其他模块只引用本文档，不重复维护供应商地址、请求协议和 API key 规则。

## 二、模型调用统一口径

- 模型唯一调用来源为 `https://platform.aimom.net/`。
- 请求格式采用 OpenAI-compatible Chat Completions。
- 只有服务端可以调用模型供应商，前端和管理端页面不得直接调用模型供应商。
- API key 只通过服务端环境变量配置，不在管理端保存或展示，也不保存到数据库。
- `ai_models.provider_model_id` 是实际传给供应商 `model` 字段的调用 ID，用户端只展示模型对外名称。
- 用户端可选模型必须由服务端远端返回的启用模型列表决定，前端不得写死模型列表。
- AI 调用成功后的真实 usage 是对话、编辑器分析等 AI 能力扣费的唯一依据。
- 若后续需要更换供应商地址、请求协议、鉴权方式或 usage 解析口径，必须先更新本节，再同步检查引用本节的业务模块文档。

## 三、功能范围

| 编号   | 功能             | 描述                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- |
| **AM1** | 模型列表         | 展示模型对外名称、平台调用 ID、状态、排序、上下文、最大输出和计费倍率。 |
| **AM2** | 新增模型         | 支持新增模型基础信息、评分、上下文能力、温度参数和计费倍率。         |
| **AM3** | 编辑模型         | 支持修改模型展示信息、评分、上下文能力、温度参数、排序和计费倍率。   |
| **AM4** | 启用/停用模型    | 支持启用或停用模型；停用后用户端不能继续选择，但不影响历史流水。     |
| **AM5** | 计费倍率配置     | 支持直接填写倍率，也支持按成本价和目标加价率快速生成倍率。           |
| **AM6** | 默认模型协同     | 系统配置页可从启用模型中选择编辑器 AI 检查默认模型。                 |
| **AM7** | 模型筛选         | 支持按状态、评分范围、上下文范围和最大输出范围筛选模型。             |

筛选项：

- 状态：`active`、`inactive`。
- 逻辑评分、文笔评分、知识面评分范围。
- 最大上下文 token 范围。
- 最大输出 token 范围。

## 四、模型字段

| 字段                   | 说明                                                         |
| :--------------------- | :----------------------------------------------------------- |
| 模型对外名称           | 用户端展示的模型名称，例如 `DeepSeek-v4-pro`。               |
| 平台调用 ID            | 在 `platform.aimom.net` 中实际传给 `model` 字段的 ID。       |
| 模型描述               | 给用户和管理员看的模型适用场景说明。                         |
| 逻辑评分               | 整数 1-5，表示推理、结构和一致性能力。                       |
| 文笔评分               | 整数 1-5，表示写作表达和文学性能力。                         |
| 知识面评分             | 整数 1-5，表示通识和资料覆盖能力。                           |
| 最大上下文             | 模型可支持的最大上下文 token 数。                            |
| 最大输出               | 单次请求允许配置的最大输出 token 数。                        |
| temperature            | 温度参数，范围 0-2，默认 0.7。                               |
| 缓存命中输入倍率       | `prompt_cache_hit_tokens` 的计费倍率。                       |
| 缓存未命中输入倍率     | `prompt_cache_miss_tokens` 的计费倍率。                      |
| 输出倍率               | `completion_tokens` 的计费倍率。                             |
| 状态                   | `active` / `inactive`。                                      |
| 排序值                 | 用于用户端和管理端列表排序。                                 |

## 五、计费倍率规则

模型计费基准价为 `10 元 / 100万 token`。管理端支持两种配置方式：

1. **直接配置倍率**
   管理员直接填写缓存命中输入倍率、缓存未命中输入倍率和输出倍率。

2. **成本价 + 目标加价率快速生成**
   管理员填写每 100 万 token 成本价和目标加价率后，系统按以下公式生成倍率：

   ```text
   倍率 = 成本价 * (1 + 加价率) / 10
   ```

   生成结果向上取整到 2 位小数。生成后的倍率仍允许管理员手动调整。

## 六、初始模型建议

| 模型 | 缓存命中输入倍率 | 缓存未命中输入倍率 | 输出倍率 | 说明 |
| :--- | :--------------- | :----------------- | :------- | :--- |
| `DeepSeek-v4-flash` | `0.11` | `0.11` | `0.22` | 输入成本 `1 元 / 100万 token`、输出成本 `2 元 / 100万 token`，按 10% 加价率生成。 |
| `DeepSeek-v4-pro` | `0.02` | `1.32` | `2.64` | 缓存命中输入成本 `0.10 元 / 100万 token`，未命中输入成本 `12 元 / 100万 token`，输出成本 `24 元 / 100万 token`，按 10% 加价率生成。 |

## 七、MVP 不做

- 不支持多模型供应商。
- 不在管理端维护 API key。
- 不支持模型调用健康检测。
- 不支持模型成本利润报表。
- 不支持对不同用户配置不同可用模型。

## 八、验收标准

1. 管理员可新增、编辑、启用和停用模型。
2. 模型评分只能填写 1-5 的整数。
3. temperature 只能填写 0-2 范围内的数值，默认值为 0.7。
4. 管理员可直接填写计费倍率。
5. 管理员可通过成本价和目标加价率快速生成计费倍率，生成结果向上取整到 2 位小数。
6. 停用模型后，用户端智能对话不再展示该模型。
7. 停用模型不影响历史积分流水中的模型快照展示。
8. 模型供应商地址、请求协议、API key 管理、调用 ID 和 usage 解析口径在本文档集中维护，其他模块不得重复定义不同口径。
9. 管理员可按状态、评分范围、上下文范围和最大输出范围筛选模型。
