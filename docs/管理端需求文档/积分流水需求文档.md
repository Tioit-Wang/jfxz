# 金番写作 MVP - 积分流水需求文档

## 一、目标

积分流水模块用于记录和查看用户积分发放、消耗、清零、退款和调整记录。

MVP 阶段：

- 管理端展示完整流水，用于排查账务、模型扣费和支付权益发放。
- 用户端展示简化流水，只让用户理解自己的积分变化。

## 二、功能范围

| 编号 | 功能 | 描述 |
| :--- | :--- | :--- |
| **AL1** | 管理端流水列表 | 展示流水时间、用户、变更类型、来源类型、余额类型、积分变动和变动后余额。 |
| **AL2** | AI 消耗信息 | AI 消耗流水展示模型、token usage、成本价快照、盈利倍率快照、扣费积分和关联作品。 |
| **AL3** | 流水筛选 | 管理端支持按用户、余额类型、变更类型、来源类型、模型、作品、积分变动范围和时间范围筛选。 |
| **AL4** | 流水详情 | 管理端查看单条流水的来源 ID、模型快照、usage、成本价快照、盈利倍率快照和计算结果。 |
| **AL5** | 用户端简化流水 | 用户可查看自己的流水，字段只包含时间、积分变动或消耗积分、关联作品。 |

## 三、管理端列表字段

| 字段 | 说明 |
| :--- | :--- |
| 流水时间 | 流水创建时间。 |
| 用户 | 用户邮箱或昵称。 |
| 余额类型 | `vip_daily`、`credit_pack`。 |
| 变更类型 | `grant`、`consume`、`expire`、`refund`、`adjust`。 |
| 来源类型 | 例如 `plan_vip_daily`、`plan_bundled_credit_pack`、`credit_pack`、`ai_chat`、`editor_check`、`system_adjust`。 |
| 来源 ID | 对应订单、订阅、会话、作品或其他业务来源 ID。 |
| 关联作品 | AI 对话和编辑器分析流水展示作品名称或作品 ID；非作品流水可为空。 |
| 模型 | AI 消耗流水展示模型名称快照；非 AI 流水可为空。 |
| 缓存命中输入 token | AI 消耗流水展示。 |
| 缓存未命中输入 token | AI 消耗流水展示。 |
| 输出 token | AI 消耗流水展示。 |
| 积分变动 | 正数为增加，负数为扣减，保留 2 位小数。 |
| 变动后余额 | 该条流水写入后的用户总积分余额，保留 2 位小数。 |

## 四、详情字段

AI 消耗流水详情至少展示：

- 模型 ID。
- 模型名称快照。
- 平台调用 ID 快照。
- 关联作品。
- 缓存命中输入 token。
- 缓存未命中输入 token。
- 输出 token。
- 输入成本价快照。
- 缓存命中输入成本价快照。
- 输出成本价快照。
- 盈利倍率快照。
- 积分汇率快照。
- 扣费积分。
- 变动后余额。

权益发放流水详情至少展示：

- 订单 ID。
- 商品名称快照。
- 商品类型。
- 锁定的权益快照。
- 发放积分。
- 变动后余额。

## 五、用户端简化流水

用户端流水只展示当前用户自己的记录。

| 字段 | 说明 |
| :--- | :--- |
| 时间 | 流水创建时间。 |
| 积分变动 | 正数为发放或调整增加，负数为 AI 消耗或清零扣减。 |
| 关联作品 | AI 对话和编辑器分析展示作品名称；非作品流水可为空或展示“系统”。 |

用户端不展示：

- token 明细。
- 模型倍率。
- 模型调用 ID。
- 成本信息。
- 后台来源 ID。
- 管理端排查字段。

## 六、业务规则

- 流水为账务记录，管理端只读，不支持手动新增、编辑或删除流水。
- VIP 每日积分发放生成 `grant` 流水，来源类型为 `plan_vip_daily`。
- VIP 每日积分在次日 5:00 清零未使用部分，生成 `expire` 流水。
- 套餐附带积分权益包发放生成 `grant` 流水，来源类型为 `plan_bundled_credit_pack`。
- 独立积分权益包购买发放生成 `grant` 流水，来源类型为 `credit_pack`。
- AI 调用前必须完成余额预检；余额不足时不生成消费流水。
- AI 调用成功后按真实 OpenAI-compatible usage 生成消费流水。
- AI 调用失败、解析失败、模型未配置、模型停用或无有效结果时不生成消费流水。
- 一次 AI 调用可生成多条消费流水：先扣 `vip_daily`，不足部分再扣 `credit_pack`。
- 正常扣费流程不允许扣成负数。
- 历史流水使用模型名称、平台调用 ID、成本价快照和盈利倍率快照展示，不受模型后续改名、改成本价、改盈利倍率或停用影响。

## 七、MVP 不做

- 不支持管理端手动补流水。
- 不支持流水导出。
- 不支持成本利润统计报表。
- 不支持按流水发起退款。
- 用户端不展示完整排查明细。

## 八、验收标准

1. 管理员可查看所有用户的积分流水。
2. 管理员可按用户、余额类型、变更类型、来源类型、模型、作品、积分变动范围和时间范围筛选流水。
3. AI 消耗流水可展示模型快照、token usage、成本价快照、盈利倍率快照、关联作品和扣费积分。
4. VIP 每日积分清零可生成清零流水。
5. 用户端可查看简化流水，字段只包含时间、积分变动或消耗积分、关联作品。
6. 用户端不展示 token、模型成本价、盈利倍率、模型调用 ID、成本或后台来源 ID。
