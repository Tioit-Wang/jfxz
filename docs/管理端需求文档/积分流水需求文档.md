# 金番写作 MVP - 积分流水需求文档

## 一、目标

管理端积分流水模块用于只读查看用户积分发放、消耗、过期、退款、调整和系统赠送补平记录。

MVP 阶段积分流水只在管理端展示，用户端暂不提供流水明细页面。

## 二、功能范围

| 编号   | 功能             | 描述                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- |
| **AL1** | 流水列表         | 展示积分流水时间、用户、变更类型、来源类型、积分变动和变动后余额。   |
| **AL2** | AI 消耗信息      | AI 消耗流水展示模型、token usage、倍率快照和扣费积分。               |
| **AL3** | 系统赠送补平     | 展示因 AI 超额扣费产生的系统赠送补平流水。                           |
| **AL4** | 流水筛选         | 支持按用户、变更类型、来源类型、模型和时间范围筛选。                 |
| **AL5** | 流水详情         | 查看单条流水的来源 ID、模型快照、usage、倍率快照和计算结果。         |

## 三、列表字段

| 字段 | 说明 |
| :--- | :--- |
| 流水时间 | 流水创建时间。 |
| 用户 | 用户邮箱或昵称。 |
| 变更类型 | `grant`、`consume`、`expire`、`refund`、`adjust`、`system_grant`。 |
| 来源类型 | 例如 `plan_monthly`、`plan_bundled_topup`、`topup_pack`、`ai_chat`、`editor_check`、`system_overage_grant`。 |
| 来源 ID | 对应订单、会话、作品或其他业务来源 ID。 |
| 模型 | AI 消耗流水展示模型名称快照；非 AI 流水可为空。 |
| 缓存命中输入 token | AI 消耗流水展示。 |
| 缓存未命中输入 token | AI 消耗流水展示。 |
| 输出 token | AI 消耗流水展示。 |
| 积分变动 | 正数为增加，负数为扣减，保留 2 位小数。 |
| 变动后余额 | 该条流水写入后的用户总积分余额，保留 2 位小数。 |

## 四、详情字段

AI 消耗流水详情至少展示：

- 模型 ID。
- 模型名称快照。
- 平台调用 ID 快照。
- 缓存命中输入 token。
- 缓存未命中输入 token。
- 输出 token。
- 缓存命中输入倍率快照。
- 缓存未命中输入倍率快照。
- 输出倍率快照。
- 扣费积分。
- 变动后余额。

系统赠送补平流水详情至少展示：

- 来源消费流水 ID。
- 补平前总余额。
- 系统赠送积分。
- 补平后总余额。

## 五、业务规则

- 流水为账务记录，管理端只读，不支持手动新增、编辑或删除。
- AI 调用成功后按真实 OpenAI-compatible usage 生成消费流水。
- AI 调用失败、解析失败、模型未配置或无有效结果时不生成消费流水。
- 一次 AI 调用可生成多条消费流水：先扣月度积分，不足部分再扣加油包积分。
- 若 AI 消费导致加油包积分临时为负数，系统追加一条 `system_grant` 流水补平到 0。
- `system_grant` 流水需关联导致超额的消费流水。
- 历史流水使用模型名称、平台调用 ID 和倍率快照展示，不受模型后续改名、改倍率或停用影响。

## 六、MVP 不做

- 不在用户端展示流水明细。
- 不支持管理端手动补流水。
- 不支持流水导出。
- 不支持成本利润统计报表。
- 不支持按流水发起退款。

## 七、验收标准

1. 管理员可查看所有用户的积分流水。
2. 管理员可按用户、变更类型、来源类型、模型和时间范围筛选流水。
3. AI 消耗流水可展示模型快照、token usage、倍率快照和扣费积分。
4. 系统赠送补平流水可追溯到对应超额消费流水。
5. 用户端不展示积分流水明细。
