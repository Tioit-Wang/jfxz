# 金番写作 MVP - 管理端概述

**定位**：管理端是内部后台，用于支撑用户、模型、套餐、订单订阅、积分流水和会话的最小管理需求。  
**原则**：后端尽量少做功能，优先提供查看、启停和基础状态管理能力。

## 一、管理端范围

MVP 管理端包含以下模块：

- 用户管理
- 模型管理
- 套餐管理
- 订单与订阅管理
- 积分流水
- 会话管理
- 系统配置

## 二、管理端原则

- 优先做查看和状态控制，不做复杂运营工具
- 所有管理端列表页都提供与当前列表字段匹配的基础筛选项
- 不做复杂报表、财务结算和自动化运营
- 不做细粒度权限系统，MVP 只区分 `admin` 和 `user`

## 三、MVP 不做

- 数据分析看板
- 手动改写用户会话内容
- 手动新增、编辑或删除积分流水
- 复杂审核流
- 复杂财务后台
