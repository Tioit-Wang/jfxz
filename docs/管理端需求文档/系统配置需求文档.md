# 金番写作 MVP - 系统配置需求文档

## 一、目标

管理端系统配置模块用于维护全局配置项，支持不同类型字段，并为支付等基础能力提供统一配置入口。

## 二、功能范围

| 编号   | 功能             | 描述                                                                 |
| :----- | :--------------- | :------------------------------------------------------------------- |
| **AC1** | 配置列表         | 展示配置分组、配置键、类型、当前值、是否必填。                       |
| **AC2** | 配置编辑         | 支持按字段类型编辑配置值。                                           |
| **AC3** | 分组查看         | 支持按配置分组查看，例如 `payment.alipay_f2f`。                      |
| **AC4** | 密文字段展示     | `secret` 类型字段默认掩码显示。                                      |
| **AC5** | 配置初始化       | 系统初始提供默认 seed 配置项，值默认为空。                           |
| **AC6** | AI 检查模型配置  | 支持为编辑器 AI 检查选择默认模型。                                   |
| **AC7** | 配置筛选         | 支持按配置分组、值类型、是否必填和是否为空筛选配置项。               |

筛选项：

- 配置分组。
- 值类型：`string`、`integer`、`decimal`、`boolean`、`json`、`secret`。
- 是否必填。
- 是否为空。

## 三、支付宝当面付配置项

MVP 阶段管理端需要支持以下配置项：

- `payment.alipay_f2f.enabled`
- `payment.alipay_f2f.app_id`
- `payment.alipay_f2f.app_private_key`
- `payment.alipay_f2f.alipay_public_key`
- `payment.alipay_f2f.notify_url`
- `payment.alipay_f2f.seller_id`
- `payment.alipay_f2f.timeout_express`

## 四、积分汇率配置项

MVP 阶段管理端需要支持以下配置项：

- `billing.points_per_cny`

规则：

- 该配置项用于定义人民币与积分的汇率，即 1 元人民币等于多少积分。
- 默认值为 `10000`。
- 积分汇率影响所有模型的积分售价计算和商品定价参考。
- 修改积分汇率不影响已购买订单的权益（已购权益以订单快照为准）。

## 五、AI 配置项

模型基础信息和计费倍率由模型管理模块维护，系统配置页只维护跨模块使用的默认模型选择。

MVP 阶段管理端需要支持以下配置项：

- `ai.editor_check.model_id`

规则：

- 该配置项用于指定编辑器 AI 检查默认模型。
- 可选值必须来自状态为 `active` 的模型记录。
- 用户端编辑器 AI 检查不提供模型切换入口。
- 模型 API key 来自服务端环境变量，不在系统配置页维护。

## 六、MVP 不做

- 配置版本回滚
- 配置发布审批流
- 多环境配置管理
- 配置变更审计报表
- 在系统配置页维护模型详情和计费倍率
