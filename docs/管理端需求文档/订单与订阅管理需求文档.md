# 妙蛙写作 MVP - 订单与订阅管理需求文档

## 一、目标

管理端订单与订阅模块用于查看购买记录和当前订阅情况。

## 二、功能范围

| 编号   | 功能           | 描述                                                                 |
| :----- | :------------- | :------------------------------------------------------------------- |
| **AO1** | 订单列表       | 展示订单编号、用户、商品类型、商品名称、金额、状态、支付时间。       |
| **AO2** | 订单详情       | 查看订单基础信息、商品权益快照和积分发放来源。                       |
| **AO3** | 订阅列表       | 展示用户当前订阅套餐、状态、开始时间、结束时间和续费顺序。           |
| **AO4** | 订阅详情       | 查看某条订阅的基础信息、锁定的每日 VIP 积分、套餐时长和关联订单。    |
| **AO5** | 支付记录查看   | 查看订单关联的支付记录、支付渠道状态、二维码创建情况和通知结果。     |
| **AO6** | 订单筛选       | 支持按用户、商品类型、订单状态、支付渠道、金额范围和下单/支付时间范围筛选订单。 |
| **AO7** | 订阅筛选       | 支持按用户、套餐、订阅状态、开始时间范围、结束时间范围和续费顺序筛选订阅。 |

筛选项：

- 订单列表：用户、商品类型、订单状态、支付渠道、金额范围、下单时间范围、支付时间范围。
- 订阅列表：用户、套餐、订阅状态、开始时间范围、结束时间范围、续费顺序。

订阅展示规则：

- MVP 支持续费顺延，不支持升级、降级或立即切换权益。
- 续费订单形成新的订阅周期，开始时间为当前最后一个未到期订阅的结束时间。
- 订阅权益以购买时锁定的快照为准，不受后续套餐配置修改影响。

详细支付方案见 `../支付接入需求文档.md`。

## 三、MVP 不做

- 后台手动收款
- 手动补单流程
- 财务结算报表
- 自动退款处理
