# 部署文档

## 架构概览

**三层架构（1Panel 管理）：**

```
互联网 → goodgua.net:443
        ↓
1Panel OpenResty (宿主机，SSL 终结)
        ↓
goodgua-nginx (:18081) — Nginx Alpine，路径路由
  ├── /api/* → goodgua-backend:8000 (strip /api prefix)
  └── /*     → goodgua-frontend:3000
```

- **OpenResty** → `127.0.0.1:18081` 转发给 Nginx 容器
- **Nginx 容器** (`goodgua-nginx`)：路径路由，`/api/` 重写前缀后转发后端
- **前端容器** (`goodgua-frontend`)：Next.js standalone output，不暴露宿主机端口
- **后端容器** (`goodgua-backend`)：FastAPI + Uvicorn，连 `app-network` 和 `1panel-network`
- **PostgreSQL**：1Panel 内置管理，通过 `1panel-network` 容器名互通

## 容器服务

| 服务 | 镜像 | 端口映射 | 网络 |
|------|------|---------|------|
| nginx | nginx:alpine | `127.0.0.1:18081:80` | app-network |
| frontend | 构建自 `./frontend` | 无（内部 3000） | app-network |
| backend | 构建自 `./backend` | 无（内部 8000） | app-network, 1panel-network |

Nginx 只监听 `127.0.0.1:18081`，不直接暴露到公网。前端和后端不映射宿主机端口，仅通过内部 Docker 网络通信。

## 关键配置说明

### 前端

- `next.config.ts` 设 `output: "standalone"` — Next.js 独立部署模式，生成自包含的 server.js
- 多阶段 Docker 构建：builder 阶段编译，runner 阶段只复制 `.next/standalone` 产物
- 以非 root 用户 `nextjs` 运行
- 安全头：CSP、HSTS（生产）、X-Frame-Options、X-Content-Type-Options 等
- `NEXT_PUBLIC_API_BASE_URL` 通过 Docker build arg 注入

### 后端

- FastAPI + Uvicorn，2 个 worker
- 启动参数：`--proxy-headers --forwarded-allow-ips *`（信任 Nginx 反代转发的客户端 IP）
- Pydantic Settings 读取 `GOODGUA_` 前缀的环境变量
- 生产环境强制校验：JWT >= 32 字节、禁止 CORS 通配符、禁用支付模拟器
- 日志写入容器内 `/app/logs/`（rotating file handler）
- 时区设为 `Asia/Shanghai`

### Nginx

- 路径路由：`/api/` 剥离前缀后转发后端，其他请求转发前端
- 设置 `X-Real-IP`、`X-Forwarded-For`、`X-Forwarded-Proto` 头
- WebSocket 支持（`Upgrade`/`Connection` 头）
- 上传限制 `client_max_body_size 10M`

### 数据库

- PostgreSQL 由 1Panel 管理，不在 docker-compose.yml 中定义
- 后端通过 `1panel-network`（external network）连接 PostgreSQL 容器
- 连接字符串格式：`postgresql+asyncpg://user:pass@postgres:5432/dbname`

## 环境变量

生产环境变量在 **1Panel 应用商店**中配置，参考 `.env.production.example`：

| 变量 | 说明 |
|------|------|
| `NEXT_PUBLIC_API_BASE_URL` | 前端 API 地址，生产为 `https://goodgua.net/api` |
| `GOODGUA_ENV` | 运行环境，固定 `production` |
| `GOODGUA_DATABASE_URL` | PostgreSQL 连接字符串（asyncpg） |
| `GOODGUA_JWT_SECRET` | JWT 签名密钥，至少 32 字节 |
| `GOODGUA_CORS_ORIGINS` | 允许的跨域来源，生产不能含 `*` |
| `GOODGUA_TRUSTED_PROXY_IPS` | 信任的反代 IP 段，逗号分隔 |
| `GOODGUA_BOOTSTRAP_ADMIN_EMAIL` | 首次启动自动创建的管理员邮箱 |
| `GOODGUA_BOOTSTRAP_ADMIN_PASSWORD` | 首次启动自动创建的管理员密码 |
| `GOODGUA_AI_PROVIDER_BASE_URL` | AI 接口地址 |
| `GOODGUA_AI_PROVIDER_API_KEY` | AI 接口密钥 |
| `GOODGUA_USER_SESSION_SECONDS` | 用户会话时长（默认 86400） |
| `GOODGUA_ADMIN_SESSION_SECONDS` | 管理员会话时长（默认 7200） |

## 部署命令

在服务器项目目录下：

```bash
docker compose up -d --build
```

首次部署前需确保：
1. 1Panel 中已创建 PostgreSQL 数据库
2. `1panel-network` 已存在（`docker network ls | grep 1panel-network`）
3. `.env.production.example` 中的变量已在 1Panel 中配置

## 更新部署

```bash
git pull
docker compose up -d --build
```

Docker Compose 会自动重建有变化的镜像并重启容器。

## 日志查看

```bash
# 所有服务
docker compose logs -f

# 指定服务
docker compose logs -f backend
docker compose logs -f nginx
docker compose logs -f frontend
```
